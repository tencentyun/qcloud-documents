## 功能描述
通过线下对接配置回调业务 URL 的方式，通知业务方事件下发的状态。


## 请求
### 请求包体

| 属性        | 类型     | 必选   | 说明     |
| --------- | ------ | ---- | ------ |
| appId     | String | 是   | 应用 ID  |
| callId    | String | 是   | 呼叫 ID  |
| data      | String | 否   | 用户透传数据 |
| event     | String | 是   | 通知事件类型（playoverrpt） |
| timeStamp | String | 否   | 时间戳    |

## 响应
### 响应包体

| 属性 | 类型   | 必选 | 说明                                                         |
| ---- | ------ | ---- | ------------------------------------------------------------ |
| code | String | 是 | 响应码，000000 表示成功<br>错误原因请参见 [错误码](https://cloud.tencent.com/document/product/679/14513) |
| msg  | String | 是 | 结果说明                                                     |

## 实际示例
### JSON 请求示例

```
{
    "appId": "247e35ff320a4142a105024055c367cf", 
    "callId": "88-2018031000182918181271218-108bc381ca0f42d8-out", 
    "data": "", 
    "event": "playoverrpt", 
    "timeStamp": "20160131170852107"
}
```

### JSON 响应示例

```
{
    "code": "000000", 
    "msg": "success"
}
```
