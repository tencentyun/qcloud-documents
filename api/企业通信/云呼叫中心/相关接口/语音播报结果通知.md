## 功能描述
通过线下对接配置回调业务 URL 的方式，通知业务方事件下发的状态。

### 请求包体

| 属性           | 类型     | 必选   | 说明                                       |
| ------------ | ------ | ---- | ---------------------------------------- |
| appId        | String | 是   | 应用 ID                                    |
| callId       | String | 是   | 呼叫 ID                                    |
| event			   |String	|	是 |通知事件类（callnotifyrpt）
| ansCode	     |int	    |是	 |0：呼叫失败-有可能是媒体服务器出错，建议上层重呼<br>1：振铃<br>2：接听（成功接听）<br>3：通话中（号码正在通话）<br>4：ASR超时 – （废弃）<br>5：不在服务区<br>6：欠费/停机<br>7：被叫拒接<br>8：被叫关机<br>9：空号<br>10：停机（废弃）<br>11：线路错误<br>12：呼叫超时 – 系统挂机<br>13：超限（8分钟3次，24小时8次）<br>14：线路超时未返回<br>15：超限（主叫超限，需要换号码呼叫）<br>16：线路繁忙- 稍后再呼<br>91：TTS 转换错误<br>92：呼叫媒体错误，需要重试<br>93：呼叫超时，需要重试<br>99：其他
| dtmf	       |String  |	否	|请求参数 getKey 为 1 时，上报用户的按键信息，<br>用户超时未按键为 timeOut|
| timeStamp    | String | 否   | 时间戳                                      |

## 响应
### 响应包体
| 属性   | 类型     | 必选   | 说明                 |
| ---- | ------ | ---- | ------------------ |
| code | String | 是   | 响应码，000000表示成功<br>错误原因：详细参见 [错误码](https://cloud.tencent.com/document/product/679/14513) |
| msg  | String | 否   | 结果说明              |

## 实际示例
### JSON 请求示例
```
{
    "appId": "247e35ff320a4142a105024055c367cf", 
    "callId": "6-20190118161244121247412-43fbab286806000-out", 
    "event": "callnotifyrpt", 
    "dtmf": "timeOut", 
    "ansCode": 2
}
```

### JSON 响应示例
```
{
    "code": "000000", 
    "msg": "success"
}
```
