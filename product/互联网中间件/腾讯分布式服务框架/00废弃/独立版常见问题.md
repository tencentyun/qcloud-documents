### 应用相关

1. ### 新建应用时，为什么提示“应用名称已经存在，请更换其他名称”，或新建部署组时，提示“部署组名称已存在，请更换其他名称”？

   目前在 TSF 平台上，每个租户下面的应用名称不能重复。尽管在您的控制台列表上可能看不到同名应用，但在其他项目下可能有人已经使用了同名应用，需要您在起名时避免相同名称。同理，在同一个命名空间下，部署组名称不能重复，需要您在为部署组起名时避免相同名称的部署组。

2. ### TSF 是否支持在同一台服务器上安装多个应用？

   在 TSF 中，应用的部署有两种类型：

   - CVM 云主机独占实例：在一台 CVM 云主机 上，仅部署单独一个应用。通常根据应用需要的资源配置来购买 CVM 云主机。
   - 容器实例：TSF 使用 Docker 容器在一台独立的 CVM 云主机 上创建多个 Docker 实例，允许在每一个 Docker 实例上部署一个应用。

3. ### TSF 应用实例状态为什么显示 Agent 异常？

   TSF Agent 会定期上报心跳数据给 TSF 管理模块，如果 Agent 停止上报状态，则某段时间后该机器将会被判定为未知状态。通常而言，该问题是由于 Agent 停止导致。

   用户可尝试在云主机界面，重启该云主机。
