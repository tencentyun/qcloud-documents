VPC 对等连接是一种跨 VPC 互访的网络互联服务，使得两个 VPC 私有网络可以通过私网 IP 直接访问对方，就像它们属于同一网络一样。黑石 VPC 对等连接，可以支持以下互联场景，以便您构建灵活、弹性的云网络环境。

- 同账号/跨账号 VPC 私有网络互联
- 同地域/跨地域 VPC 私有网络互联
- 黑石-黑石 VPC 私有网络互联
- 黑石-公有云 VPC 私有网络互联

## 黑石 VPC 对等连接规格
### 黑石对等连接数量

| 类型 |数量 | 
|---------|---------|
| 每个 VPC | 4 | 
| 黑石-黑石（同可用区） | 3 | 

如需更多数量对等连接请咨询售后或者解决方案架构师。

### 黑石-黑石对等连接带宽

| 类型 | 带宽 | 
|---------|---------|
| 同可用区 | 无上限 | 
| 同地域 | 1Gbps| 
|跨地域|10Mbps/20Mbps/50Mbps/100Mbps/200Mbps/500Mbps/1Gbps |

如需更大带宽请咨询售后或者解决方案架构师。

### 黑石-公有云对等连接带宽

| 类型 |带宽 | 
|---------|---------|
| 同地域 |1Gbps | 
| 跨地域 | 10Mbps/20Mbps/50Mbps/100Mbps/200Mbps/500Mbps/1Gbps | 

同地域默认 1Gbps，上限 10Gbps，如需更大带宽请咨询售后或者解决方案架构师。

## 黑石 VPC 对等连接使用限制
 黑石 VPC 对等连接存在以下限制：
- 对等连接的两端 VPC CIDR 不可以重叠。
- 一个 VPC 的多个对等连接网络间 CIDR 不可重叠。
- **对等连接互通性不传递**
对等连接使 VPC 私有网络之间两两建立互联，但是这种互通关系不发生传递。例如，如下图所示，VPC 1 与 VPC 2 建立了对等连接，VPC 1 和 VPC 3 也建立了对等连接。然而由于对等连接的不传递性，VPC 2 和 VPC 3 的流量不能互通。
![](https://main.qcloudimg.com/raw/7c3b1548374fe88b3e3e49f79f1cf7b9)
- **黑石-公有云 VPC 对等连接路由设置**
建立黑石-公有云 VPC 对等连接后，需要在公有云 VPC 路由表中设置相应路由策略指向对等连接，黑石 VPC 路由表自动学习对等连接路由无需设置。
- 跨地域对等连接费用由对等连接申请方支付。
- 若对方不接受对等连接请求，申请将在 7 天后自动过期失效。

## 黑石 VPC 对等连接监控与告警
黑石 VPC 对等连接监控指标包括：

| 监控指标 |单位 | 
|---------|---------|
|    出带宽        |    Mbps       |
|    入带宽        |    Mbps       |
|    出包量        |    pps       |
|    入包量        |    pps       |
|    延时        |   ms        |

黑石 VPC 对等连接告警策略包括：

| 告警指标或事件 |告警策略 | 
|---------|---------|
|    出带宽       |    用户设置阈值周期性告警       |
|    入带宽        |    用户设置阈值周期性告警       |
|    出包量        |    用户设置阈值周期性告警       |
|    入包量        |    用户设置阈值周期性告警      |
|    对等连接中断       |  事件发生后周期性告警        |
