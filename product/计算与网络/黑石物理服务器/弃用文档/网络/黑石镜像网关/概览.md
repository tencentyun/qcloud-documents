传统 IDC，通常使用流量镜像或者分光的方式，将进出 IDC 的流量复制出来做分析。如下图所示：
![](https://main.qcloudimg.com/raw/3f3e62691380bdc87c2e3bde6f9475b6.png)

但是在云数据中心，由于流量走向具有不确定性（为了满足弹性业务需求，通常会有多个物理路径，按需调度），以及多租户的网络流量混合，传统的分光方案无法实施。

针对此类客户诉求，黑石推出流量镜像网关产品，通过专有的网络方案满足此类需求。其工作原理描述如下：
![](https://main.qcloudimg.com/raw/260ac8333edabfae1e1a23c934079c1a.png)

- 此处的黑石镜像网关完成流量分拣（保证用户只能接收到自己的归属流量），然后以ERSPAN（Encapsulated Remote Switch Port Analyzer）的方式发送给位于用户 VPC 内的黑石物理机。
- 因此，用户在黑石网络中设置流量镜像网关时，需要在创建镜像网关之后设置相应的 SPAN 规则，指定需要获取哪些公网流量（非本用户所有的公网流量无法获取镜像流量）。

### 规格

| 规格| 带宽上限 | 
|---------|---------|
| 大 | 300Mbps | 
| 中 | 1000Mbps | 
| 小 | 4000Mbps | 

### 使用约束
- 带宽限速以镜像网关发送给镜像流量接收机的流量带宽为准（包含公网出流量+公网入流量，并且计入 ERSPAN 报文的 GRE 头部字节），超出规格上限部分流量被静默丢弃。
- 黑石镜像网关提供流量监控，用户可以在云监控上查看相关数据。
- 镜像网关只能拉取镜像网关所在地域的公网流量。
- 镜像网关不支持规格变更。
- 黑石镜像网关创建时会占用用户 VPC 的 IP 地址，系统自动创建/27的子网，用户无法在控制台上操作此子网。
- 一个镜像网关可以绑定多个公网 IP。一个公网 IP 只能与一个镜像网关绑定。
