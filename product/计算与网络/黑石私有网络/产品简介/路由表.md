路由表由一系列路由规则组成，用于控制私有网络（VPC）内出流量走向。如下图所示，黑石上的路由表只有默认路由表一种类型，每一个私有网络对应一个默认路由表。
路由表由一系列路由策略组成，路由策略包括路由目的端、下一跳类型和下一跳组成，黑石私有网络路由表用户可配置的下一跳类型仅为物理服务器和 IPsec VPN 网关，其它类型的路由策略由系统自动配置。
![](https://main.qcloudimg.com/raw/7aee14fd34c1b3928dfeb1611cef3fce.png)

>!黑石私有网络路由表中指向 NAT 网关、公网网关、专线通道和运维 SSL VPN 的路由并不支持在路由表中显示。

## 默认路由表
用户创建私有网络时，系统会自动为其生成一个默认路由表。可以在默认路由表中添加、删除路由规则，但无法删除该默认路由表。


## 路由规则
路由规则用来控制数据包的路由途径，包括自定义路由规则和系统自动产生的路由规则两种类型，其中自定义路由规则包含了三个参数：
- 目的端：目的网段描述（仅支持网段格式，如果希望目的端为单个 IP，可设置掩码为32（如：172.16.1.1/32），目的端不可以是路由表所在私有网络内的 IP 段。
- 下一跳类型：私有网络的数据包出口，自定义路由下一跳类型支持“物理服务器”、“IPsec VPN 网关”。
- 下一跳：指定具体跳转至哪个下一跳实例（物理服务器填写 IP 地址，IPsec VPN 使用下一跳 ID 标识）。

## 自动产生的路由规则
- 默认 local 路由规则，含义为私有网络内网互通，其路由规则为 [ Local，Local，Local ]，该路由规则不能被删除和修改。
- 对等连接路由规则，当用户创建对等连接后，目的 IP 为对端私有网络 CIDR 的路由规则将自动加入到默认路由表，该路由不能禁用且不能编辑。
- 云联网路由规则，当用户将私有网络绑定到云联网实例后，云联网实例中路由将自动加入到默认路由表，该路由可以禁用和启用但不能编辑。

>!云联网中路由加入到黑石私有网络默认路由表将作特殊处理，目的端是私有网络的子网路由将汇聚成两条路由，目的端是专线网关的路由不作处理。

## 路由规则优先级
当路由表中存在多条路由规则时，路由优先级由高至低分别为：
- 私有网络内流量：私有网络内流量最优先匹配。
- 最精确路由：非私有网络内流量根据最精确路由规则匹配。

>!用户配置0.0.0.0/0默认路由的前提条件是：私有网络没有启用了 EIP 或者 NAT 网关。因为 EIP 和 NAT 网关会自动在路由表中配置0.0.0.0/0默认路由（在路由表中不支持显示），会与用户配置的0.0.0.0/0默认路由冲突造成业务流量中断。

## 使用约束
- 每个 VPC 关联一个默认路由表。
- 默认路由表不能删除。
- 默认 local 路由规则不能删除。
- 不支持 BGP 和 OSPF 等动态路由协议。
- 目的网络不支持以下 IP 地址：
 -  私有网络 CIDR 内的子网
 - 内部服务地址：9.0.0.0/8，100.64.0.0/10，169.254.0.0/16
- 如下表所示，每个 VPC 上创建的路由表和在每个路由表中添加的路由规则均存在数量限制：

| 资源             | 限制（个） |
| -------------- | ----- |
| 每个私有网络内的路由表个数  | 1     |
| 每个路由表的自定义路由策略数 | 200   |

## 计费模式
路由表的创建和使用完全免费。
