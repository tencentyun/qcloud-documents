## 操作场景

本文档指导您如何格式化并挂载数据盘。您可以根据实际需求，选择不同的方式进行操作：
- [手动格式化并挂载数据盘](#ManualFormat)
- [自动格式化并挂载数据盘](#AutoFormat)

## 注意事项

- 请在格式化之前，确保数据盘中没有数据或已对重要数据进行备份。格式化后，数据盘中的数据将被全部清空。
- 为避免服务发生异常，请在格式化之前，确保云服务器已停止对外服务。

## 前提条件

已登录腾讯云云服务器。

## 手动格式化并挂载数据盘

>! 执行下述命令时，请注意修改数据盘盘符。以下操作以数据盘盘符为 `vdb` 为例。
>

#### 格式化数据盘

>! 在进行分区格式化时，开发者可以自行决定文件系统的格式，如`ext3`、`ext4`等。以下操作以 `ext4` 格式为例。
>
执行以下命令，将数据盘进行格式化。
```
mkfs.ext4 /dev/vdb
```

#### 挂载数据盘

1. 执行以下命令，创建挂载点 data 目录。
```
mkdir /data
```
2. 执行以下命令，挂载新分区。
```
mount /dev/vdb /data
```
3. 执行以下命令，验证数据盘是否挂载成功。
```
df -h
```
返回类似如下信息则说明挂载成功，即 Linux 服务器已经挂载数据盘。
```
Filesystem      Size  Used Avail Use% Mounted on
/dev/vdb        50G  53M   47G   1% /data
```

#### 实现开机自动挂载

>! 当您希望云服务器在重启或开机时能自动挂载数据盘，必须将数据盘挂载信息添加到 `/etc/fstab` 中，实现开机自动挂载。否则云服务器重启或开机后，不会自动挂载数据盘。
>
1. 执行以下命令，添加分区信息。
```
echo '/dev/vdb /data ext4 defaults 0 0' >> /etc/fstab
```
2. 执行以下命令，查看分区信息。
```
cat /etc/fstab
```
返回类似如下信息则说明添加数据盘挂载信息成功。
```
/dev/vdb /data ext4 defaults 0 0
```

## 自动格式化并挂载数据盘

在腾讯云的 Linux 服务器上，您可以通过运行以下 Shell 脚本来实现自动格式化和挂载数据盘。具体操作步骤如下：
1. 执行以下命令，创建 .sh 文件。以下操作以创建 `automountData.sh` 文件为例。
```
touch automountData.sh 
```
2. 执行以下命令，编辑 `automountData.sh` 文件。
```
vim automountData.sh
```
3. 将以下内容拷贝至 `automountData.sh` 文件中，并保存文件。
```
#!/bin/bash
type=ext4
mount_dir=/data
mkfs.$type /dev/vdb 
mkdir -p $mount_dir
echo "/dev/vdb $mount_dir $type defaults 0 0" >> /etc/fstab
mount -a
```
4. 执行以下命令，设置对 `automountData.sh` 文件的操作权限。
```
chmod 777 automountData.sh
```
5. 执行以下命令，运行 `automountData.sh` 文件，即可实现自动格式化和挂载数据盘。
```
./automountData.sh
```
