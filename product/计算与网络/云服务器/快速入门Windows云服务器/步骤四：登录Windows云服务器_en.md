1) For the CVM for which you have purchased public bandwidth/traffic, you can log in to Windows CVM from your local Windows machine (remote connection MSTSC):

On the local Windows machine, click "Start" - "Run", enter "mstsc" command to open the Remote Desktop Connection dialog box.

In the input box, input the public IP of Windows Server (Log in to [CVM Console](https://console.qcloud.com/cvm) to check the public IP of CVM), as shown below:
![](//mccdn.qcloud.com/img56b1a11a3c31f.png)

Click "Connect", then enter the account ID (Administrator) and the initial password from the internal message (or the password modified by you) in the opened page to log in.
![](//mccdn.qcloud.com/img56b1a28d6bb8d.png)

Click on the arrow icon after the password input box to log in to Windows CVM. 

2) You can log in to the CVM from console whether you have purchased public bandwidth/traffic or not and regardless of the local operating system. In the Action column of CVM list, click "Log In" button to connect to Windows CVM via VNC.

![](//mccdn.qcloud.com/img56b1a6cb7b3e8.png)

By sending the Ctrl+Alt+Del command at the top left corner, enter the system login screen:

![](//mccdn.qcloud.com/img56b1a6ff2e305.png)

Enter the account ID (Administrator) and the initial password from the internal message (or the password modified by you) to log in.

> NOTE: This terminal is exclusive, that is, only one user can log in using the console at a time.
