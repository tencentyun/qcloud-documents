腾讯云CVM实例的生命周期是指 CVM 实例从启动到销毁所经历的状态。通过使用腾讯云云服务器从启动到销毁期间对实例进行合理的管理，可确保运行于云服务器上的应用程序高效经济地提供服务。

下图显示实例状态之间的转换。有关包年包月类型实例和按量计费类型实例的更多内容，可以参考 [计费模式说明](/doc/product/213/2180)。

![](//mc.qcloudimg.com/static/img/551771757a0419084585ccbfad776d86/image.png)

CVM实例有以下状态：
- 创建中 
- 运行中 
- 重启中
- 关机中 
- 已关机 
- 重置中
- 销毁中 
- 已销毁 
- （可选）已回收


## 实例启动
执行启动实例操作后实例将进入启动中（pending）状态。启动时指定的 [实例类型](/document/product/213/7153) 将决定实例的硬件配置，系统将使用在启动时指定的镜像来启动实例，且可能要过一小段时间才能连接到实例，当实例准备就绪后将进入运行中（running）状态。

实例会获得一个私有 IP 地址，相同地域相同网络环境内（VPC或基础网络）的其他云服务器可用其与之通信；若选择了公网带宽/流量的实例还会获得一个公网 IP 地址，可用于从 Internet 与该实例通信。用户可以连接到正在运行的实例，然后像使用普通的计算机一样使用它。

更多信息请参考[购买并启动实例](/doc/product/213/4855)、[登录 Windows 实例](/doc/product/213/5435) 和 [登录 Linux 实例](/doc/product/213/5436)。

## 实例重启
我们建议用户任意选择腾讯云控制台、腾讯云云 API 来重新启动实例，而非在实例中运行操作系统重启命令。执行重启实例操作后实例将进入重启中状态。

重启实例相当于重启计算机，重启后实例仍保留其公有 IP 地址、私有 IP 地址以及其硬盘上的所有数据。完成重启通常需要花费几十秒至几分钟的时间，该时间具体取决于实例配置。

更多信息请参阅[重启实例](/doc/product/213/4928)。

## 实例关机

当用户某一时刻需要停止服务时，可以对实例执行关机操作。用户可以任意选择使用腾讯云控制台或腾讯云云 API 来关机实例。

关机后的云服务器实例仍会在控制台中显示。<font color="red">不管是否关机，按量计费类型实例会持续收费直至用户销毁它。</font>包年包月类型实例到期后的一段时间内也会呈现关机状态并移至回收站中。需要注意的是，用户 ***无法*** 连接已关机的实例。

关机实例相当于关闭计算机，关机是某些操作的前提（如调整配置、重置密码等）。关机操作本身并不会改变云服务器的公有 IP 地址、私有 IP 地址及其硬盘上的所有数据，但后续操作（调整硬件配置等）可能导致 IP 的改变，请用户参考相应操作说明。

更多信息请参阅[关机实例](/doc/product/213/4929)。

## 实例销毁
用户不再需要云服务器实例时，可以终止该实例。其中：
- 不再需要包年包月类型实例时，实例到期后进入回收站，并在七天后自动销毁。
- 不再需要按量计费类型实例时，需要用户手动销毁该实例，可通过腾讯云控制台或腾讯云云 API 实现。

实例销毁时实例的系统盘及购买时指定的数据盘会随之销毁，但挂载在其上的弹性云盘不受影响。

更多信息请参阅[销毁实例](/doc/product/213/4930)。
