单个租户下 API 网关的使用限制如下：

| 类别                           | 限制  |
| ------------------------------ | ----- |
|单租户下的服务数量	  | 50个  |
|每个服务中的 API 数量	  | 100个  |
|每个服务的自定义独立域名数	  | 5个  |
|单租户下使用计划数	  | 200个  |
|单租户下密钥对数	  | 400对  |
|每个使用计划可绑定的密钥数	  | 50个  |
|单个密钥对可被绑定的使用计划数	  | 10个  |
|单个使用计划可配置的最大 QPS	  | 2000  |
|使用计划可绑定的 API 数量	  | 10000个  |
|单个服务可支持被访问的最大 QPS	  | 500  |
|单次请求上传文件大小	  | 16MB  |
|后端对接 SCF 时，单次请求大小	  | 6MB  |
|单个服务下可配置的应用数总和	  | 1000个  |
|单个API绑定的最大应用数	  | 1000个  |
|单个插件可绑定的服务数	  | 50个  |
|单租户可创建的API文档数	  | 200个  |

>?
- 对于 API 网关共享集群，当用户服务受到攻击时，可能有被下线的风险。如需保证服务稳定运行，建议选购专享实例。
- 对于 API 网关共享集群，单个服务可支持被访问的最大 QPS 为500，API 网关共享集群不能提升数额。
- 如果您有更高的 QPS 需求，请参见 [专享实例计费](https://cloud.tencent.com/document/product/628/43549) 文档购买专享实例，其中QPS 可支持超过50000。
- 通过 API 网关上传文件的相关指引，请参见 [上传文件](https://cloud.tencent.com/document/product/628/49883)。
