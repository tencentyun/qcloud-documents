Tencent Cloud provides Cloud Monitor feature for all users by default, so you do not need to enable it manually. When you use CCS, Cloud Monitor can help you collect relevant monitoring data.
You can view the monitoring data of CCS in the following ways:
## Viewing Cluster Monitoring Information
1. Log in to the [CCS Console](https://console.cloud.tencent.com/ccs), and click "Cluster" in the left navigator bar.
2. Click the "Monitor" icon under the cluster whose monitoring data is to be viewed, and the page of cluster monitoring information is displayed.
![](//mc.qcloudimg.com/static/img/d7d889d66e0af6b30753a5dc2cb0e365/image.png)

## Viewing Service Monitoring Information
1. Log in to the [CCS Console](https://console.cloud.tencent.com/ccs), and click "Service" in the left navigator bar.
2. Click the "Monitor" icon under the service whose monitoring data is to be viewed, and the page of cluster monitoring information is displayed.
![](//mc.qcloudimg.com/static/img/83d02d21727f206650e1b1238786ae93/image.png)

## Viewing Pod Monitoring Information
1. Log in to the [CCS Console](https://console.cloud.tencent.com/ccs), click "Service" in the left navigator bar, and then click the **Name** of the service (such as nginx in the figure) whose monitoring data is to be viewed in the service list.
![](//mc.qcloudimg.com/static/img/a6886023547972e9e6561112047cc126/image.png)
2. Click the "Monitor" icon under the pod whose monitoring data is to be viewed in the pod list, and the page of pod monitoring information is displayed.
![](//mc.qcloudimg.com/static/img/fcd40438f1102832a4c9fae5cc7d096f/image.png)

## Acquiring Monitoring Data Through API
Users can use the API "GetMonitorData" to acquire the monitoring data of all products. For more information, please see [Read Monitoring Data](/doc/api/248/4667).
