## 概述
云原生 etcd 支持通过控制台管理实例数据，用户可直接在控制台增删改查数据。

## 前提条件
- 已 [创建 etcd 集群](https://cloud.tencent.com/document/product/457/58178)。
- 具有集群的 ForwardEtcd 操作权限，管理员可在[策略管理](https://console.cloud.tencent.com/cam/policy)中为用户授予该权限，详见：[授权用户在控制台管理数据和账号](./账号授权.md@授权用户在控制台管理数据和账号)。

## 数据管理
> 云原生 etcd 控制台使用 etcd API V3与实例交互，暂不支持 API V2。
> etcd API V3没有目录概念，控制台显示的目录为渲染后的效果，真实 key 值应为完整路径。
### 查看数据
控制台以key中的 ‘/’ 为标记，树状分层展示数据
> 为了防止过量的数据加载影响用户使用，将默认限制仅拉取1000条数据(limit=1000),因此控制台默认展示的数据不能保证完整性。

![](https://qcloudimg.tencent-cloud.cn/raw/5aebd27ebb968b5774f35c30fb6a73bc.png)

用户可在搜索框中，以标签格式使用指定的搜索条件筛选数据：
![](https://qcloudimg.tencent-cloud.cn/raw/912f425c883d19868db40e05a90b9d61.png)
| 字段 | 说明 |
| -- | -- |
| key | 查询指定名称的 key，**需要输入完整的key，而不是控制台显示的子路径** |
| range_end | 范围查询 [key,range_end) 的结束点。注意，**完整的 range 查询需要输入 key,range_end 两个标签字段的值**，如果没有设置key，只设置了range_end， 则查询 Keys < range_end ，如果设置了key 但 range_end 为“\0”，则范围为所有keys >= Key |
| prefix | 以指定字符开头的key |
| revision | revision etcd的数据版本，每当数据发送变更后，revision 值都会+1。如果revision值小于或等于零，则查询最新的键值。如果已被压缩，则查询返回 ErrCompacted 。|
| min_create_revision | 查询创建版本大于该 revision 值的 key 。|
| max_create_revision | 查询创建版本的小于该 revision 值的 key 。|
| min_mode_revision | 查询修改版本大于该 revision 值的 key |
| max_mode_revision | 查询修改版本小于该 revision 值的 key 。|

更多关于查询参数的介绍，详见 [RangeRequest](https://etcd.io/docs/v3.5/dev-guide/api_reference_v3/#message-rangerequest-apietcdserverpbrpcproto)。

### 新增数据
1. 可直接点击【新建】 创建数据，或在对应的目录中点击 ’+‘ 增加数据：
![](https://qcloudimg.tencent-cloud.cn/raw/d65cebd61e386e225b4d3b626c8aa268.png)
2. 在弹窗中，输入 key 和 value 值。控制台除支持原始数据'RAW'格式以外，还支持’json‘、’XML‘两种格式渲染与格式校验。
![](https://qcloudimg.tencent-cloud.cn/raw/084b04ae9a2fc8d60b0164fe9226b17b.png)
3. 点击确定后，控制台将自动刷新并展示数据。

### 修改数据
1. 在数据窗口，点击右上方的编辑按钮
![](https://qcloudimg.tencent-cloud.cn/raw/1bf8cd1bde6531b7bcb449231aa402a6.png)
2. 在弹窗中，修改数据，并点击确认。
