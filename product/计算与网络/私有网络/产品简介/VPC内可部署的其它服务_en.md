
Tencent Cloud VPC can be integrated with many other Tencent Cloud Services. The relationships between them are shown in the following table:

## 1. Compute and Network Services

| Product | Relationship with VPC | 
|---------|---------|
| [Cloud Virtual Machine (CVM)](https://www.qcloud.com/product/cvm.html) | You can deploy the CVM in the VPC |
| [Cloud Load Balance (CLB)](https://www.qcloud.com/product/clb.html?idx=2) | You can deploy private network LB or public network LB in a private network |
| [NAT Gateway](https://www.qcloud.com/product/nat.html) | It can be used to access the Internet with traffic in the VPC|
| [Cross-region Connection (CRC)](https://www.qcloud.com/product/crc.html) | It can be used to connect the VPCs in multiple regions |
| [VPN Connections](https://www.qcloud.com/product/vpn.html) | Connect VPC and user's IDC via IPsec VPN |
| [Direct Connect (DC)](https://www.qcloud.com/product/dc.html) | Connect VPC and user's IDC via Direct Connect |
| [Basic Cloud Monitor (BCM)](https://www.qcloud.com/product/bcm.html) | Provide custom alarms for the key metrics of certain services in the VPC |

## 2. Database Services
| Product | Relationship with VPC | 
|---------|---------|
| [CDB for MySQL](https://www.qcloud.com/product/cdb.html) | You can deploy the database service in the VPC. This service will consume the IP resources of VPC |
| [CDB for SQL Server](https://www.qcloud.com/product/sqlserver.html) | Same as above |
| [CDB for TDSQL](https://www.qcloud.com/product/cdb.html) | Same as above |
| [CDB for PostgreSQL](https://www.qcloud.com/product/postgresql.html) | Same as above |
| [Cloud Cache Service Memcached](https://www.qcloud.com/product/cmem.html) | Same as above |
| [Cloud MongoDB Service](https://www.qcloud.com/product/mongodb.html) | Same as above |
| [Cloud HBase Service](https://www.qcloud.com/product/HBase.html) | Same as above |
