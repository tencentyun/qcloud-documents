Web 函数基于函数内置的标准语言镜像环境中，您必须创建一个可执行文件 bootstrap 来启动您的 Web Server，然后将该文件和您的代码文件一起打包部署，完成 Web 函数创建。实际处理请求时，您的 Web Server 通过监听指定的 `9000` 端口接收到所有的 HTTP 请求，并转发给后端服务完成逻辑处理并返回给用户。

### 启动文件作用
 Bootstrap 为 Web Server 的启动文件，保证您的 Web 服务正常启动并监听请求。除此之外，您还可以根据需要在 Bootstrap 中自定义实现更多个性化操作：

- 设定运行时依赖库的路径及环境变量等。
- 加载自定义语言及版本依赖的库文件及扩展程序等，如仍有依赖文件需要实时拉取，可下载至 /tmp 目录。
- 解析函数文件，并执行函数调用前所需的全局操作或初始化程序（如开发工具包客户端 HTTP CLIENT 等初始化、数据库连接池创建等），便于调用阶段复用。
- 启动安全、监控等插件。

### 使用前提
- 需具有可执行权限，请确保您的 Bootstrap 文件具备 777 或 755 权限，否则会因为权限不足而无法执行
- 能够在 SCF 系统环境（CentOS 7.6）中运行。
- 如果启动命令文件是shell脚本，第一行必须有 `#!bin/bash` 

### 创建方式
#### 本地打包上传
您可以本地编写您的 Bootstrap 启动文件，确保文件权限满足要求后，和项目代码一起打包部署在 Web 函数上。
#### 控制台快速创建
您也可以在控制台创建流程"高级配置"模块编辑您的启动文件，SCF 云函数为常用 Web 框架提供了通用启用模版，您也可以根据实际情况进行修改，创建完成后，控制台将自动把您的代码和 Bootstrap 一起打包部署。

### 常见 Web Server 启动命令模版
- Express.js

```shell
#!bin/bash
export PORT=9000
npm run start
```

- Koa.js

```shell
#!bin/bash
export PORT=9000
npm start
```

- Egg.js

```shell
#!bin/bash
export PORT=9000
export EGG_SERVER_ENV=prod
npx egg-scripts start
```

- Next.js
```shell
#!bin/bash
export PORT=9000
npx next start --p $PORT
```

- Nuxt.js

```shell
#!bin/bash
export PORT=9000
npx nuxt start --hostname 0.0.0.0 --port $PORT
```

- Flask

```shell
#!bin/bash
export PORT=9000
python app.py # 改为您自己的启动文件名
```
