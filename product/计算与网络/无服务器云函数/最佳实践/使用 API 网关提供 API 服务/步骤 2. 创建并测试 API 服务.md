## 操作场景
在此部分中，将创建一个 API 网关中的服务和相关的 API 规则，对接在步骤一中创建的 SCF 云函数，并通过控制台的 API 测试，来测试 API 的正确性。
>! API 服务和函数必须位于同一个地域下。在本教程中，将使用北京区域来创建 API 服务。
> 

## 创建 API 服务及规则
1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/apigateway)，从云产品中选择【互联网中间件】-【API 网关】。
2. <span id="step2">单击【服务】选项卡，并切换地域为【北京】。</span>
3. 单击【新建】按钮以新建 API 服务，在弹出窗口中写入服务名 `blogAPI`，单击【提交】创建。
4. 完成服务创建后，进入创建的 `blogAPI` 服务，选择【API 管理】选项卡。
5. <span id="step5">单击【新建】创建 API，路径为  `/article`，请求方法为 GET，为了方便后面的测试，勾选免鉴权，无需输入参数配置，单击【下一步】。</span>
6. 后端类型选择为【cloud function】，选择函数为步骤一中创建的 `blogArticle`，单击【完成】。
7. <span id="step7">再次在【API管理】选项卡中单击【新建】创建 API，路径为 `/article/{articleId}`，请求方法为 GET，勾选上免鉴权，参数配置中输入名称为 `articleId` 的参数，参数位置为 Path，类型为 int，默认值为 1，单击【下一步】。</span>
8. 后端类型选择为【cloud function】，选择函数为步骤一中创建的 `blogArticle`，单击【完成】。

## 调试 API 规则
1. 针对前面 [第 5 步](#step5) 创建的 `/article` API，单击【API 调试】，在调试页面发送请求，确保返回结果内的响应 Body，为如下内容：
```
[{"category": "blog", "time": "2017-12-05 13:45", "id": 1, "title": "hello world"}, {"category": "blog", "time": "2017-12-06 08:22", "id": 2, "title": "record info"}, {"category": "python", "time": "2017-12-06 18:32", "id": 3, "title": "python study"}]
```
2. 针对前面 [第 7 步](#step7) 创建的 `/article/{articleId}` API，单击【API 调试】，在调试页面将请求参数修改为 1 后发送请求，确保返回结果内的响应 Body，为如下内容：
```
{"category": "blog", "content": "first blog! hello world!", "time": "2017-12-05 13:45", "id": 1, "title": "hello world"}
```
3. 也可以修改 [第 2 步](#step2) 中的请求参数 articleId 的值为其他数字，查看响应内容。
