当专线连接不能满足您的业务需求时，您可以将网络架构平滑迁移至云联网上，即可体验安全、稳定、极速、灵活的全球互联网络，实现全网互通。

## 背景信息
用户本地 IDC 通过物理专线使用传统专用通道上云，现需要迁移至云联网，以实现和其他 VPC 的全网互联。
![](https://main.qcloudimg.com/raw/3556771fac29f3cb579a0c140de202a7.png)
## 操作步骤
<span id ="step1"></span>
1. 创建云联网专线网关，具体步骤请参见 [创建专线网关](https://cloud.tencent.com/document/product/216/19256)。
 >!所创建云联网专线网关的地域需保证与物理专线的专线接入点同地域。
2. 申请云联网类型的专用通道，并关联 [步骤1](#step1) 创建的云联网专线网关，具体操作请参见 [申请通道](https://cloud.tencent.com/document/product/216/19250)。
3. 在 [专线网关列表页](https://console.cloud.tencent.com/vpc/dcgw?rid=1) ，单击 [步骤1](#step1) 创建的云联网网关。
4. 在"IDC 网段"页签中添加用户 IDC 网段，本示例为 “192.168.0.0/24” 。
>?
>- 您可选择静态添加 IDC 网段，也可以选择自动学习 IDC 网段，自动学习功能需 [提交工单](https://console.cloud.tencent.com/workorder/category?level1_id=517&level2_id=727&source=0&data_title=%E5%85%B6%E4%BB%96%E8%85%BE%E8%AE%AF%E4%BA%91%E4%BA%A7%E5%93%81&level3_id=728&radio_title=%E5%8A%9F%E8%83%BD%E5%92%A8%E8%AF%A2&queue=3026&scene_code=17783&step=2) 进行申请。
>- 当前自动学习 IDC 网段时延为1分钟，若不能满足需求，请优先选择静态方式添加 IDC 网段。
>
![](https://main.qcloudimg.com/raw/91ee51c43216435bb57aba406d37ef44.png)
5. 新建一个云联网实例（如果已有，则不需要重新创建），并将其与专线连接接入的 VPC 关联，具体操作请参见 [新建云联网实例](https://cloud.tencent.com/document/product/877/18752)。
6. 切换 VPC 往 IDC 方向的流量路径。
  1. 在 [VPC 列表](https://console.cloud.tencent.com/vpc/vpc?rid=1) 中，单击目标 VPC 实例 ID。
  2. 在 VPC 的路由详情页面，可查看 VPC 路由的基本信息及路由策略。
  >?VPC 具备自动学习云联网路由表的能力，后加入的等价路由默认不启用。VPC 往 IDC 方向的流量路径依旧选择旧的专用通道。
  >
![](https://main.qcloudimg.com/raw/d66b405f7c4fbcdb52894f969f1ce684.png)
  3. 根据实际情况，选择以下任意一个场景进行平滑迁移。
	   - 场景一：云联网自动下发的路由与专线网关路由未产生冲突，则云联网路由生效。具体步骤请参见 [场景一](#1)。
	   - 场景二：专线网关路由包含云联网自动下发的路由，则云联网路由默认无效。具体操作请参见 [场景二](#2)。
	   - 场景三：云联网自动下发的路由包含或等于专线网关的路由，则云联网路由默认无效。具体操作请参见 [场景三](#3)。
7. 切换 IDC 往 VPC 方向的流量路径。
	1. 将云联网专线网关关联至云联网，具体操作请参见 [关联网络实例](https://cloud.tencent.com/document/product/877/18747)。
	2. 进入云联网实例的路由表，查看路由情况。
>?
>- 专用通道是静态路由，IDC 往 VPC 方向的流量若想切换至云联网通道路径，只需用户 IDC 路由指向新的子接口云联网通道即可。
>- 若专用通道均是 BGP 路由，云端接入设备的旧通道向 IDC 发的是 CIDR 大网段，而新的云联网通道向 IDC 发的是子网明细路由（目的端为子网 CIDR 的路由），根据路由掩码最长匹配原则，IDC 往 VPC 方向的流量会自动切换到云联网专用通道。
>- 您也可以禁用或启用云联网里面 VPC 或专线网关路由，来控制 IDC 往 VPC 的流量路径。
>
 ![](https://main.qcloudimg.com/raw/798100f24553136b1f6474693accabe8.png)

## 平滑迁移场景
<span id ="1"></span>
#### 场景一：云联网自动下发的路由与专线网关路由未产生冲突，则云联网路由生效。
![](https://main.qcloudimg.com/raw/2755a1c4e4cf557cc2f144dda0d5a266.png)
具体操作步骤如下： 
1. “停用”下一跳为专线网关的路由策略。
2. 查看专线网关网络流量监控数据，具体操作请参见 [查看监控信息](https://cloud.tencent.com/document/product/216/19253)，若无流量从专线网关通过，即可删除专线网关。

<span id ="2"></span>
#### 场景二：专线网关路由包含云联网自动下发的路由，则云联网路由默认无效。
由于路由根据最长掩码匹配原则转发，手动开启云联网路由即可使其生效。
![](https://main.qcloudimg.com/raw/2328b2705e8324a9348bd84fa917f272.png)
具体操作步骤如下： 
1. “启用”下一跳为云联网的路由策略。
2. 查看专线网关网络流量监控数据，具体操作请参见 [查看监控信息](https://cloud.tencent.com/document/product/216/19253)，若无流量从专线网关通过，即可删除专线网关。

<span id ="3"></span>
#### 场景三：云联网自动下发的路由包含或等于专线网关的路由，则云联网路由默认无效。
由于路由根据最长掩码匹配原则转发，您需要先调整专线网关路由策略目的段，再手动开启云联网路由即可使其生效。
![](https://main.qcloudimg.com/raw/0e1439be24c5bcacfd68d4c3a440e19d.png)
具体步骤如下：
1. “启用”下一跳为云联网的路由策略。
2. 更改专线网关的路由策略，将目的端网段调整为包含原目的端网段，且子网掩码小于云联网默认下发路由的网段。例如将“192.168.0.0/24”修改为“192.168.0.0/16”。
3. 手动启用云联网相关路由。调整后，流量将优先从下一跳为云联网的路由转发，原下一跳为专线网关的路由策略失效。
4. 查看专线网关网络流量监控数据，具体操作请参见 [查看监控信息](https://cloud.tencent.com/document/product/216/19253)，若无流量从专线网关通过，即可删除专线网关。
