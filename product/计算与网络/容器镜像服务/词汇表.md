### 实例
用户在指定地域可购买的独享的容器镜像托管服务，后台服务及核心数据存储与其他用户的实例独立。在 docker 原生概念中，实例可等同于独立的 Docker Registry 服务，即可以认为用户在云端购买并部署了私有的 Docker Registry 服务。

### 访问域名
每个容器镜像服务实例均具有专有的访问域名，用户可使用 docker 客户端指定该访问域名以登录实例并拉取，推送镜像。

### 访问凭证
用于登录实例的用户名及密码组合，其中用户名与腾讯云用户的UIN相同，密码为仅在一定时间内有效的临时密码，默认为生成后24小时内有效。

### 命名空间
实例内的目录层级，对应于 DockerHub 中的 Organization，以及 Harbor 中的 Project，可用于隔离不同项目，团队等组织内的镜像仓库。命名空间介于实例及仓库之间，其访问级别属性将直接决定内部镜像仓库的公开，私有属性。镜像仓库及 Helm Chart 仓库可公用命名空间。

### 镜像仓库
镜像仓库 (Repository) 用于存放 Docker 镜像，单个镜像仓库可对应单个具体的 docker 应用，并托管该应用的不同版本，即对应仓库内不同版本的容器镜像(docker image)。

### 触发器
用户对指定的镜像仓库及 Helm Chart 进行如推送镜像等操作时，将触发指定操作，如访问指定域名并传递产生触发的镜像仓库的基本信息。
