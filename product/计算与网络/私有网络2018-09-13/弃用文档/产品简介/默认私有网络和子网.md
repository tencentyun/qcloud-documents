默认 VPC 及默认子网功能目前处于**灰度**阶段，只有**2017.6.13后**新注册的部分账号的网络属性为 only-vpc，其它账号的网络属性均为 classic。您可以通过 [DescribeAccountVpcAttributes](https://cloud.tencent.com/document/api/215/9499) 接口查询账号的网络属性。
- **only-vpc**
账号仅支持在 VPC 下创建实例（如 CVM、LB 等），无法在 [基础网络](https://cloud.tencent.com/document/product/215/535) 内创建实例。
- **classic**
账号可在基础网络或 VPC 下创建实例。

## 默认 VPC 和默认子网
only-vpc 账号支持默认 VPC 及默认子网特性，可在每个地域创建默认 VPC 及默认子网。
默认 VPC 及默认子网能为您带来如下好处：
- 您可以在没有 VPC 或子网的情况下，完成 CVM 等实例的创建，系统会为您创建默认 VPC 及子网，为您节省了解 VPC 及子网的功能细节的时间。
- 在创建 CVM 等实例时，您可以不指定 VPC 或子网，系统将在默认 VPC 的默认子网内创建实例。

>**注意：**
>**2017.08.03 起新上线地域**对所有用户均不再支持基础网络，新上线地域可创建默认 VPC 和默认子网。

## 默认 VPC 基本信息
- 您可以像使用非默认 VPC 一样使用默认 VPC，可在默认 VPC 内添加子网、修改路由表、添加 VPN 连接等。
- 您可以创建额外 VPC，默认 VPC 不占用您在某个地域下 VPC 配额。
- 您可像使用其它 VPC 或子网一样使用默认 VPC 或默认子网，功能上没有区别。
- 默认 VPC 和默认子网可被删除。

## 在默认 VPC 内启动实例（CVM、TDB等）
### 没有默认 VPC
如果您的账号在某地域内没有默认 VPC，在购买 CVM 并创建实例时，私有网络选择 Default-VPC（ 默认），子网选择 Default-Subnet（默认）。
1. 系统将在您选择的地域为您创建一个 /16 的默认 VPC，在您选择的可用区内创建一个 /20 的默认子网，且在该 VPC 下创建默认路由表。
2. 将默认子网绑定该默认路由表。
3. 在默认子网内生成您创建的实例。
 
### 有默认 VPC，没有默认子网
如果您的账号在某地域内有默认 VPC，没有默认子网，在购买 CVM 并创建实例时，子网选择的 Default-Subnet（默认）。
1. 系统将在您选择的可用区内创建一个 /20 的默认子网。
2. 将默认子网绑定该 VPC 下的默认路由表。
3. 在该默认子网内生成您创建的实例。

### 有默认 VPC 及默认子网
如果您的账号在某地域内有默认 VPC 及默认子网，您可同选择非默认 VPC 和子网一般，选择默认 VPC 及默认子网。
