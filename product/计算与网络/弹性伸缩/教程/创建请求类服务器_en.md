The server accessing externally has an obvious feature of timeliness. In this case, Auto Scaling service can facilitate fast creation, deployment and scale-down of the server sending requests.
If the user knows when capacity scaling is needed, he/she can configure Auto Scaling scheduled policy in advance. By the configured time, the system will automatically increase or decrease the number of CVM instances without the need to wait, saving your deployment and instance costs.
