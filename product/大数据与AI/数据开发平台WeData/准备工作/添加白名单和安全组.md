目前云数据仓库 PostgreSQL 计算引擎和腾讯云 MySQL 数据库存在白名单/安全组的限制，因此在使用相关服务前需要对 WeData 访问 IP 开放设置。通常添加白名单/安全组涉及以下两种情况：
- 获取和使用云数据仓库 PostgreSQL 集群。如需通过其他子网中的 CVM 或者外网机器访问云数据仓库 PostgreSQL集群时，需要将这些 IP 地址加入到白名单中。
- 使用腾讯云 MySQL 数据库。如果您使用的腾讯云 MySQL 数据库高于基础版，为保证数据库的安全稳定，在开始使用数据库的实例前您需要将 WeData 访问 IP 加到目标数据库的安全组中。

## 添加云数据仓库 PostgreSQL 集群白名单
为了在工作空间中使用和访问云数据仓库 PostgreSQL 集群，您需要将下表中 WeData 的访问 IP 地址添加到云数据仓库 PostgreSQL 集群允许的白名单中。
```
152.136.100.86、81.70.41.169、81.70.41.126、81.70.43.64、81.70.21.104、81.70.40.81、152.136.103.114、152.136.148.172、81.70.41.36
```
#### 操作步骤
1. 查看集群用户名及数据库
登录 [云数据仓库 PostgreSQL](https://console.cloud.tencent.com/cdwpg) 控制台，在【集群列表】中选择目标集群，单击【管理】。
![](https://main.qcloudimg.com/raw/d630511e1a61dbbbbf5f470986d9228e.png)
2. 选择【配置】，单击【新建白名单】，进入新建白名单页面。
![](https://main.qcloudimg.com/raw/7677d7a6d850524ce37bb2f56119e118.png)
3. 在新建白名单页，填写白名单名称、指定用户、指定数据库和 IP 白名单信息。填写完成后，单击【确认】。
![](https://main.qcloudimg.com/raw/60df7783cf18ac3dbca35dd1f70a287d.png)

>?WeData 使用云数据仓库 PostgreSQL 集群默认数据库进行连通性测试，请填写默认用户名并至少对默认数据库 postgres 开通白名单。如果还需要使用集群中的其他数据库，请同时对该数据库开通白名单。默认用户名及数据库信息查看方式：
![](https://main.qcloudimg.com/raw/82517b0876c68b4f9394dc34e646b177.png)

**示例：**如果您需要在工作空间中绑定名为“workpace”的云数据仓库 PostgreSQL 集群（默认集群用户名：wedata，默认数据库名：postgres），并且使用该集群中的数据库“database1”进行数据开发。那么您需使用用户名“wedata”将默认数据库“postgres”以及“database1”开通白名单。
 ![](https://main.qcloudimg.com/raw/fc0ba595e8fd97f9ca229ca85892c940.png)
 
 
## 添加腾讯云 MySQL 数据库安全组
安全组是一种有状态的包含过滤功能的虚拟防火墙，用于设置单台或多台云数据库的网络访问控制，是腾讯云提供的重要的网络安全隔离手段。如果您使用的腾讯云 MySQL 数据库高于基础版，您需要将以下 IP 加到目标数据库的安全组中。具体操作参见 [管理云数据库安全组](https://cloud.tencent.com/document/product/236/9537)。
```
152.136.100.86、81.70.41.169、81.70.41.126、81.70.43.64、81.70.21.104、81.70.40.81、152.136.103.114、152.136.148.172、81.70.41.36
```
