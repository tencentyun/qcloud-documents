本文为您详细介绍在 CODING 中如何进行缺陷管理。

## 进入项目

1. 登录 [CODING 控制台](https://console.cloud.tencent.com/coding)，单击【立即使用】进入 CODING 使用页面。
2. 单击页面右上角的 <img src ="https://main.qcloudimg.com/raw/d94a8e60dd3a41d0af07d72ae0e9d70e.png" style ="margin:0">，进入项目列表页面，单击项目图标进入目标项目。
3. 单击左侧菜单栏中的【项目协同】>【缺陷】，进入缺陷管理页面。

## 功能介绍

在进行缺陷管理操作时，您可以设置缺陷的所属类型。CODING 缺陷管理默认提供了五种缺陷类型：

- 功能缺陷
- UI 界面问题
- 易用性问题
- 安全问题
- 性能问题

![](https://main.qcloudimg.com/raw/505eb7cfb42b4f5f61bcb755e6c4d02d.png)

您可以根据团队需求自定义缺陷类型。仅团队所有者及管理员可创建缺陷类型。缺陷类型添加成功后，团队内所有项目均可选用该缺陷类型。

## 创建缺陷类型

1. 前往【首页】>左侧菜单处【功能设置】>【项目协同设置】>【缺陷类型】，单击末端操作按钮。
![1](https://main.qcloudimg.com/raw/0737ce0a04b522ccf1f80b9374578c1b.png)
2. 输入所需【缺陷类型】，单击【添加】按钮或回车即可完成缺陷类型添加。
![](https://main.qcloudimg.com/raw/cb66ea99cc2bc8fbfb9efc8237259042.jpg)

### 使用缺陷类型

在任一项目【项目协同】>【缺陷】页面单击【创建缺陷】，在缺陷创建页即可选用上述操作所添加的缺陷类型。
![2](https://main.qcloudimg.com/raw/17666f2ceebe017db538d24ba8a2cec5.png)

## 管理缺陷模块

缺陷模块管理主要用于配置缺陷所属模块，且模块数据可在项目内共享。

### 创建缺陷模块

创建不同的模块可便于管理和定位缺陷位置，满足专业化场景需要。

1. 进入任一项目，点击左侧的【项目设置】，进入【项目协同】>【项目协同】>【模块设置】>【创建模块】。
![](https://main.qcloudimg.com/raw/4316cdcf7ca6b5438adc48187231f0a4.png)
2. 输入【模块名称】，单击【保存】按钮即可完成缺陷模块添加。
![](https://main.qcloudimg.com/raw/cda8f630b512aa18b91f436c93f0434e.png)

### 使用缺陷模块

在任一项目【项目协同】>【缺陷】页面单击【创建缺陷】，在缺陷创建页即可选用上述操作所添加的缺陷模块。
![](https://main.qcloudimg.com/raw/53e792fb78d8c2b1d768bff4a3f88cf5.jpg)
