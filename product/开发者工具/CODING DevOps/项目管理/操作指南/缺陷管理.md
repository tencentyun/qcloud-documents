本文为您详细介绍在 CODING 中如何进行缺陷管理。

## 进入项目
1. 登录 [CODING 控制台](https://console.cloud.tencent.com/coding)，单击【立即使用】进入 CODING 使用页面。
2. 单击页面右上角的 <img src ="https://main.qcloudimg.com/raw/d94a8e60dd3a41d0af07d72ae0e9d70e.png" style ="margin:0">，进入项目列表页面，单击项目图标进入目标项目。
3. 单击左侧菜单栏中的【缺陷管理】，进入缺陷管理列表页面。

## 缺陷处理
缺陷处理包含了对具体缺陷的创建、查看、更新、删除、评论、记录日志等操作。

### 创建缺陷
1. 在缺陷管理列表页，单击【创建缺陷】。
2. 填写缺陷标题、缺陷描述，按需设置缺陷各属性。
	 - 标题：必填项，建议取辨识度高的标题。
	 - 需求描述：输入描述内容，支持 Markdown。
3. 单击【创建】，即可创建成功。

>?
>-  缺陷可以设置优先级、截止日期、缺陷分类等基本信息，并指定处理人员处理。
- 缺陷必须处于“未开始”、“进行中”和“已完成”其中一个阶段。


### 查看缺陷列表
缺陷列表是该项目的所有缺陷的列表视图，是测试人员和缺陷处理人员的主要工作界面。用户可以迅速对缺陷进行筛选和操作。尤其当存在大量缺陷时，通过搜索与筛选配合使用，可快速筛选出用户想要找的缺陷。了解更多筛选器的使用方法请参见 [使用筛选器](https://cloud.tencent.com/document/product/1113/36837) 。

### 查看缺陷详情
缺陷详情页展示了缺陷具体内容和属性。
1. 在缺陷管理列表页，单击目标缺陷标题或缺陷引用 ID 。
2. 页面右侧将会出现缺陷详情页，在缺陷详情页可进行缺陷属性修改。

### 缺陷状态流转
1. 在缺陷详情页中，单击缺陷标题下方的状态图标，即可看到所有状态。需求默认包含5个状态，依次是：待处理、处理中、待验证、 已拒绝、已关闭。
![](https://main.qcloudimg.com/raw/bca14d6ce97a3e753f32ef1d1683769c.png)
2. “已关闭”的缺陷单可以修改状态为“重新打开”。
![](https://main.qcloudimg.com/raw/2eebc8fcbc91a58cb84bbe0a30119895.png)
3. 管理员可以根据缺陷进行的阶段选择所对应的状态。


### 删除缺陷

删除缺陷有两种途径：
- 在缺陷详情页右上角单击 <img src ="https://main.qcloudimg.com/raw/f000f0cee5005373b46f1d348ee3053f.png" style = "margin:0">，出现的下拉菜单中单击【删除】。
- 在缺陷列表页，选择对应缺陷前的复选框，然后在下方可选择执行批量删除。


## 管理缺陷类型和缺陷模块

### 创建缺陷类型
1. 企业所有者或管理员，将鼠标移动至右上角的头像图标处，单击浮现出来的【企业管理】，进入企业概览页面。
2. 单击左侧菜单栏中的【属性与流程】，进入属性配置页面。
3. 选择“缺陷类型”属性，单击末端操作按钮。
![](https://main.qcloudimg.com/raw/6a0ede122e07f80e3952421f8b0b2e2d.png)
4. 输入所需“缺陷类型”，最多12个字符。
5. 单击【添加】或回车，即可完成缺陷类型添加。

>? 
>- 在缺陷创建页即可选用上述操作所添加的缺陷类型。
- 仅企业所有者及管理员可创建缺陷类型。缺陷类型添加成功后，企业内所有项目均可选用该缺陷类型。

### 创建缺陷模块
创建不同的模块可便于管理和定位缺陷位置，满足专业化场景需要。
1. 在任一项目中，选择左侧菜单栏中的【设置】>【属性与流程】，进入属性与流程页面。
2. 单击【模块设置】，进入模块设置页面。
3. 单击右上角的【创建模块】。
4. 输入【模块名称】。
5. 单击【保存】即可完成缺陷模块添加。

>?在缺陷创建页即可选用上述操作所添加的缺陷模块。

## 缺陷统计
缺陷统计是通过对缺陷进行统计帮助团队跟踪各状态的缺陷总量，追踪缺陷变化趋势，查看成员工作量等。辅助团队决策者对项目质量和成员工作量做出判断。

### 查看缺陷统计
1. 在任一项目中，选择左侧菜单栏中的【统计】>【缺陷统计】，进入缺陷状态统计页面。
2. 缺陷状态统计，统计了当前项目下缺陷在各状态下的总量。使用条状展示便于观察缺陷处理的进度。并且列出关键指标“关闭率”、“待办数”、“缺陷综述”和“处理中数量”。
	- 关闭率：关闭状态下的缺陷总量 / 缺陷总量 x 100%
	- 待办数量：状态为待办的缺陷总量。
	- 缺陷总数：当前项目下的所有缺陷数量。
	- 处理中数量：处于处理中的缺陷总量。
3. 缺陷状态统计页面，还展示了新增趋势统计、缺陷状态趋势统计、按经办人统计和按创建人统计的折线图。
 - 趋势统计：统计每天新创建的缺陷数量，用于跟踪缺陷增长趋势。默认显示近2周的数据。
 - 缺陷状态趋势统计：统计每天缺陷在各状态下的数量，用于跟踪缺陷状态变化趋势。当天数据存在1小时延迟。默认显示近2周的数据。
 - 按经办人统计：分别统计各处理人的缺陷数量（按照缺陷处理人分组统计），并区分各状态缺陷总量。鼠标悬浮时可以看到具体数量。
 - 按创建人统计：分别统计各创建者创建的缺陷数量（按照缺陷创建人分组统计），并区分各状态缺陷总量。鼠标悬浮时可以看到具体数量。
