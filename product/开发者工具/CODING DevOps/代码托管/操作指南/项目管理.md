本文为您详细介绍项目的管理与使用。

## 操作步骤
### 创建项目
1. 登录 CODING，鼠标移动至右上角的<img src ="https://main.qcloudimg.com/raw/7531b01c25014beb2754277107fc4ab1.png" style ="margin:0">，单击浮出来的【创建项目】。
![](https://main.qcloudimg.com/raw/213ecdb0e276db77093c93ea5eb84136.png)
2. 进入项目创建页面后，填写项目地址、项目名称即可创建项目，同时您还可以填写项目描述并添加项目成员。建议您勾选“启用 README.md”文件初始化项目。
3. 单击【新建项目】完成创建项目，创建人为项目的管理员。

### 导入通用项目

1. 单击右上角的<img src ="https://main.qcloudimg.com/raw/7531b01c25014beb2754277107fc4ab1.png" style ="margin:0">，进入项目列表页面。
2. 单击右侧【导入项目】。
3. 系统会自动检测是否有同名项目。若存在同名项目系统会自动重命名项目，用户也可以单击图标自行修改。

### GitLab.com 项目导入
>?
目前 GitLab.com 导入功能暂时只支持代码仓库导入，WIKI、文件、任务、看板等模块正在开发中，请您耐心等待。
项目导入前，需要先在 CODING 的“成员管理”页面添加与导入项目所对应的项目成员，二者成员一一匹配。

1. 单击右上角的<img src ="https://main.qcloudimg.com/raw/7531b01c25014beb2754277107fc4ab1.png" style ="margin:0">，进入项目列表页面。
2. 单击右侧【导入项目】。
3. 单击 GitLab.com 右侧【开始导入】。
2. 进入 GitLab.com 的授权页面，检查账号是否登录。若未登录 GitLab.com 将跳转至登录页面，若已登录将直接跳转至授权页面。
3. 授权成功后将跳转至“导入项目”页面，勾选需要导入的项目。
4. 单击【下一步】后跳转至“检测项目冲突”页面，目的是为检查项目名是否冲突。如果项目名冲突，系统会自动修改项目名，同时用户也可以自行修改项目名（用户自行修改的项目名也需要再次检测是否重名）。
5. 单击【下一步】后跳转至“关联成员”页面，需要添加与导入项目所对应的项目成员。
6. 单击【确认导入】按钮即可成功导入项目，可在项目列表中查看到此项目。
7. 在“导入项目”页面可查看导入项目记录。

### 更新项目信息
1. 企业所有者/管理员将鼠标移动至右上角的头像图标处，单击浮出来的【企业管理】。
	![](https://main.qcloudimg.com/raw/33d18f12363223371a4b0ad9c65fe3ff.png)
2. 进入企业概览页面，单击左侧菜单栏中的【项目管理】。
3. 进入企业项目页面，选择需要更新信息的项目单击右侧的【项目设置】。
4. 在项目设置弹窗中可更新项目地址、项目名称、项目描述及项目时间。

> 注意：修改项目名称将导致项目的访问 URL（包含 Git 仓库的 URL）改变，在此之前的 URL 将失效。Git 仓库地址修改方法：`git remote set-url origin [NEW_URL] `。

### 成员管理
1. 企业所有者/管理员将鼠标移动至页面右上角的头像图标处，单击浮出来的【企业管理】。
2. 进入企业概览页面，单击左侧菜单栏中的【项目管理】。
3. 进入企业项目页面，选择对应的项目单击右侧的【成员管理】。
4. 在成员管理弹窗中可添加/移除项目成员、修改项目成员用户组。

### 删除/归档项目
1. 企业所有者/管理员将鼠标移动至页面右上角的头像图标处，单击浮出来的【企业管理】。
2. 进入企业概览页面，单击左侧菜单栏中的【项目管理】。
3. 进入企业项目页面，选择对应的项目单击最末端【…】，可进行【归档项目】或【删除项目】操作。
	- 项目归档后，项目内所有成员将无法访问该项目，且不能对该项目进行任何操作。
	- 删除项目会删除项目内的所有数据，且删除操作无法恢复。若项目内存有重要数据，建议归档项目。
![](https://main.qcloudimg.com/raw/a77aa94332236f1b5e41005d0ecf4b4c.png)
4. 被归档的项目单击【解除归档】，即可解除归档。
