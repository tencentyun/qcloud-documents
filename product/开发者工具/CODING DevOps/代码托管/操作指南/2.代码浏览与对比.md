本文为您详细介绍代码浏览与对比。

## 进入项目

1. 登录 [CODING 控制台](https://console.cloud.tencent.com/coding)，单击团队域名进入 CODING 使用页面。
2. 单击页面右上角的 <img src ="https://main.qcloudimg.com/raw/d94a8e60dd3a41d0af07d72ae0e9d70e.png" style ="margin:0">，进入项目列表页面，单击项目图标进入目标项目。
3. 选择左侧菜单【代码仓库】，进入代码仓库首页。

## 功能介绍

进入项目后，代码仓库的首页会展示项目内所有的仓库。

页面最上方显示 Git 仓库的各项主要功能，右上角可快捷克隆该 Git 仓库地址，分为 HTTPS（`https://e.coding.net/团队域名/项目名.git`）和 SSH（` git@e.coding.net:团队域名/项目名.git`）两种类型，单击地址上方的链接可切换到对应协议的仓库地址。

页面中间显示当前仓库 master 分支的一级目录结构及对应最新提交（即 push）。单击切换 master 为其他分支，即可查看其他代码分支及对应最新提交。


> !只有项目管理员、团队成员才有权限访问和操作代码，受限成员无访问权限。

![](https://main.qcloudimg.com/raw/b51ab4a471455def312a77a5aec5244a.png)



## 提交历史

单击【提交历史】，默认进入 master 分支的提交历史，提交历史按照日期倒序排列。可以切换为其他分支，查看相应的提交历史。单击提交记录或 SHA ID 进入代码版本提交详情。
![](https://main.qcloudimg.com/raw/c59070eb46c9c01d9f55f5f15394e1d2.png)

## 提交详情

进入提交详情会清晰的看见所变更的代码记录，您可以按照习惯选择合适的代码浏览模式，还可以在页面末尾发布代码行级评论。

### 代码行级对比

普通模式下，代码文件中会显示该提交改动信息 `1个文件被修改 （8行增加 / 4行减少）`，按照单个文件显示具体的提交改动。每个文件内会显示具体的行级增删修改信息，如`@@ -51,7 +51,7 @@`。其中，绿色代表增加，红色表示减少。
![](https://main.qcloudimg.com/raw/3899213d4a57133d7a091a510287cf05.png)

### 代码左右对比

单击【左右模式】，代码行级对比由普通模式切换为左右模式。用户可清晰地看出当前代码改了哪一行，修改了什么。
![](https://main.qcloudimg.com/raw/c088d6852c75148d79a877c646062afd.png)

### 高级选项

高级选项默认开启显示制表符、行内对比、代码高亮，您可以按需打开或关闭。
![](https://main.qcloudimg.com/raw/409e83e2b2277e949d9a666eca7848f9.png)

### 收起&展开

用户进入提交详情页，当改动文件数较少时，默认全部展开，当改动文件数较多时，默认全部收起。用户可单击文件名，单独展开或者收起单个文件，或者单击右侧【全部展开】和【全部收起】，展开或收起全部文件对比。
![](https://main.qcloudimg.com/raw/c04d3a6dacb26500b8403d0a7c4659e8.png)

### 代码评审

开发者可在新建合并请求时或合并请求合并前，更新标题和描述，添加其他项目成员为评审者，并关联项目内资源（如任务、文件、合并请求、Wiki）。

**评审方式**

评审者收到邀请评审通知，进入合并请求详情页面，并对代码进行评审。评审内容包括以下：
1. 该合并请求的标题、描述及关联资源是否对代码改动作出充分说明，评审者可通过评论和发起者进行沟通确认。
2. 针对提交记录对应的文件改动，进行代码行级评审（评论）。

**代码行级评审**

鼠标悬停在代码文件中的某一行，界面会显示出 “+” 号，单击即可针对该代码行发起评论。评论提交后，提交代码改动的团队成员将会收到您的评论通知，当他回复评论后，您也会收到评论通知。由此，开发者可通过代码行级评论就某一行代码进行讨论交流。
![](https://main.qcloudimg.com/raw/9992fc1d3a86ede952cb7dac84d286c1.png)

## 寻找文件

单击选择分支右侧输入框，可输入需要寻找的关键字对文件列表进行过滤。 使用 ↑ 和 ↓ 来选择文件， 单击 Enter 快速查看文件内容。 
![](https://main.qcloudimg.com/raw/e33dd35347a16388b51499075b537130.png)

## 代码文件新建 / 上传 / 下载

单击【新建】或【上传文件】，即可为当前分支新增一个一级目录代码文件，需要填写文件名、文件内容、文件提交信息。其中上传文件支持所有文件格式，单个文件不超过20M。

此处还可以勾选 为这次提交创建一个**新分支**，并发起一个合并请求选项，系统会自动以当前分支为基点创建一个新分支，并自动向当前分支提交合并请求。

单击【下载】，系统会自动以 `团队域名-项目名-分支名.zip`下载当前分支对应的 Git 仓库版本。
![](https://main.qcloudimg.com/raw/70319fedea453235e99ffd74192111a8.png)
