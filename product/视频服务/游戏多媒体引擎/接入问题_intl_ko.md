Tencent Cloud GME SDK의 사용을 환영합니다. 개발자가 Tencent Cloud GME 제품 API를 디버깅하고 연결하는 데 도움이 되도록 여기에 SDK의 FAQ와 답변을 소개합니다. 다음에서는 몇 가지 기술적인 FAQ와 답변 및 일반적인 개념에 대한 이해입니다. 본 문서는 계속 보완될 것입니다.

## 일반 문제

### GME의 구체적인 기능은 무엇입니까?

GME는 음성 채팅, 오프라인 음성 메시지 및 음성 인식 기능을 지원합니다.

### GME의 게임 시나리오에는 어떤 것이 있습니까?

E-스포츠, 커맨더 게임, 레저 게임, 마피아 게임 등입니다.

### GME는 어떤 게임 엔진과 플랫폼을 지원합니까?

GME는 현재 Unity 엔진, Unreal 엔진 및 Cocos2d 엔진과 같은 게임 엔진을 지원하며, 지원되는 플랫폼에는 Windows 플랫폼, Mac 플랫폼, iOS 플랫폼 및 Android 플랫폼 등이 있습니다.

### GME는 어떻게 시작합니까?

Tencent Cloud GME를 효과적으로 사용할 수 있도록, [액세스 가이드 문서](https://cloud.tencent.com/document/product/607/10782)가 게임 오디오 및 비디오 엔진에 대한 초급 가이드를 제공합니다.

### Android의 so 라이브러리에 armv8이 있습니까?

일시적으로 지원되지 않으며 후속 버전에서 지원될 예정입니다.

### 인증 보안을 어떻게 보장합니까?

전반적으로는 액세스 초기 단계에서 클라이언트 배포 솔루션을 사용하며, 이후에 게임 앱 백 엔드에 배포로 최적화하는 것이 좋습니다.

| 솔루션       | 장점     | 단점             |
| ---------- | -------- | ---------------- |
| 백 엔드 배포   | 안전성 높음 | 백 엔드 개발 공동 테스트 필수 |
| 클라이언트 배포 | 신속한 액세스 | 안전성 낮음         |


###  실시간 음성 및 오프라인 음성에 액세스하는 과정에서 오류 코드는 인증 실패로 나타냅니다. 어떻게 해야 합니까?
실시간 음성 및 오프라인 음성에 액세스하는 과정에서 인증 오류가 발생하면 먼저 콘솔의 오프라인 음성 서비스가 활성화되어 있는지 확인하십시오. 활성화된 경우 다음 사항을 확인하십시오.
1. SDKAppID가 콘솔과 일치 여부
2. OpenID(10000보다 커야 함)
3. 오프라인 음성 방 번호의 매개변수는 반드시 0을 입력해야 함
4. Tencent Cloud 콘솔의 키와 매칭 여부

## 공식 웹사이트 Demo 다운로드 및 수정 관련 문제

### GME Demo 및 SDK는 어디서 다운로드합니까?

[다운로드 가이드](https://cloud.tencent.com/document/product/607/18521)에서 관련 Demo 및 SDK를 다운로드하십시오. 현재 공식 웹사이트에는 Unity 엔진 Demo, Cocos2D 엔진 Demo, Android 기본 개발 Demo 및 iOS 기본 개발 Demo 등이 있습니다.

### GME Demo를 다운로드한 후 자신이 신청한 계정으로 바꾸는 것이 가능한가요?

1. 가능합니다. 콘솔에서 두 개의 번호, 즉 SDKAppID와 권한 키를 각각 얻어야 합니다.
2. 오프라인 음성을 사용해야 하는 경우 TLS 서명 키를 받아야 합니다. 상기 번호의 구체적인 획득 방법은 [액세스 가이드](https://cloud.tencent.com/document/product/607/10782)를 참조하십시오.
3. 음성 채팅에서 자신의 AppID를 사용하는 경우 AVChatViewController의 GetAuthBuffer에서 음성 채팅 키를 수정해야 합니다.

### Demo를 사용할 때 다음의 오류가 난 경우: errinfo=priv map info error.

방 입장 관련 매개변수에 오류가 있습니다. SDKAppID, 권한 키를 바꾸었는지 확인하십시오.



## 요금제 문제

### GME의 음성 채팅 서비스 요금제는 어떤 것이 있습니까? 어떻게 선택합니까?

음성 채팅 서비스는 음성 DAU 기반 요금제 및 음성 사용 시간 기반 요금제 두 가지를 지원합니다. 요금제는 선정 후에 수정이 불가합니다. 상세한 가격은 [제품 가격 문서](https://cloud.tencent.com/document/product/607/17808)를 참조하십시오.
음성 DAU 기반 요금제를 선택하면, 방을 여는 시간이 90분을 넘지 않도록 제한을 받으며, 시간을 초과한 후에는 시스템이 자동으로 방을 닫습니다. 배틀류, 캐주얼류 게임에 대해서 음성 DAU 기반 요금제를 선택하는 것이 좋습니다.
음성 시간 기반 요금제를 선택하면 방을 여는 시간에 제한이 없습니다. 소셜, VJ 호스트, 커뮤니티, 팀 배틀 등 앱에 대해서 음성 시간 기반 요금제를 선택하는 것이 좋습니다.

### 음성 DAU 기반 요금제를 선택하고, 90분을 넘긴 방은 어떻게 됩니까?

이 요금제에서 90분이 넘어간 방은 닫히며, 닫힌 후에는 콜백 함수를 받을 수 있습니다.

### GME의 음성 채팅 서비스 DAU는 어떻게 계산됩니까?

인 앱 사용자가 방에 입장하면 음성 DAU로 간주됩니다. 음성 DAU는 UserID에 따라 중복을 제거하여 다시 계산됩니다(UserID는 인 앱 사용자의 유일한 식별이며, 한 사용자는 한 개의 UserID에 대응함).

### GME의 음성 채팅 서비스는 언제부터 계산이 시작됩니까?

인 앱 사용자가 방에 입장하면 계산이 시작된다는 것으로 여겨집니다.

### GME의 음성 메시지와 텍스트 전환 서비스 요금제는 어떤 것이 있습니까? 어떻게 선택합니까?

음성 메시지 및 텍스트로 전환 서비스는 음성 메시지 DAU에 따라 요금을 계산합니다. 상세한 가격은 [제품 가격 문서](https://cloud.tencent.com/document/product/607/17808)를 참조하십시오.

### GME의 음성 메시지 및 텍스트 전환 서비스 DAU는 어떻게 계산됩니까?

인 앱 사용자가 음성 메시지를 발송하면 음성 메시지 DAU로 간주됩니다. 음성 메시지 DAU는 UserID에 따라 중복을 제거하여 다시 계산됩니다(UserID는 인 앱 사용자의 유일한 식별이며, 한 사용자는 한 개의 UserID에 대응함).



## 음성 채팅 서비스 FAQ

### 음성 채팅 서비스는 어떤 게임 시나리오를 지원합니까?

대략 다음과 같은 세 가지 종류의 시나리오를 지원합니다.
**마이크 순서 모드: **사용자가 차례대로 발언을 하고, 음질이 좋고 유창하게 말할 수 있어서 음성 마피아 게임 등의 시나리오에 적합합니다.
**자유 통화 모드: **여러 명이 동시에 말하도록 지원하고 지연이 매우 적어서 멀티 플레이어 팀플 등의 경기 게임 시나리오에 적합합니다.
**커맨드 모드: **일대다의 지휘 작전, 오디오를 통해 VJ와 게임하기 등의 시나리오 등을 대상으로 대규모 커맨더 게임에 적합합니다.

### 나에게 맞는 오디오 유형을 선택하려면 어떻게 해야 합니까?

서로 다른 응용프로그램 시나리오에 다양한 오디오 유형이 있으며 자세한 내용은 다음 리스트를 참조하십시오.

| 오디오 유형                   | 의미     | 매개변수 | 음량 유형                         | 콘솔 추천 샘플링 속도 설정                                    | 적용 시나리오                                                     |**
| -------------------------- | -------- | ---- | -------------------------------- | ------------------------------------------------------- | ------------------------------------------------------------ |
| ITMG_ROOM_TYPE_FLUENCY     | 보통 음질 | 1    | 스피커: 통화 음량. 헤드폰: 미디어 음량 | 음질에 대해 특별한 요구가 없을 경우 16K 샘플링 속도라면 충분합니다.                    | 유창성이 우선이고 지연 시간이 매우 짧은 음성 채팅으로 게임에서 팀 배틀에 적용되고 FPS, MOBA와 같은 게임에 적합합니다. |
| ITMG_ROOM_TYPE_STANDARD    | 표준 음질 | 2    | 스피커: 통화 음량. 헤드폰: 미디어 음량 | 필요한 음질에 따라 16K/48K 샘플링 속도를 선택할 수 있습니다.                 | 음질이 비교적 좋고 시간 지연이 적당하여 마피아 게임, 보드게임 등 캐쥬얼 게임의 실시간 통화 시나리오에 적합합니다. |
| ITMG_ROOM_TYPE_HIGHQUALITY | 고음질 | 3    | 스피커: 미디어 음량, 헤드폰: 미디어 음량 | 최적의 효과를 보증하기 위해서 콘솔에서 48K 샘플링 속도의 고음질 구성을 설정하기를 권장합니다. | 초고음질, 지연시간이 상대적으로 커서 음악, 댄스 등 게임 및 음성 소셜류 앱에 적용합니다. 음악방송, 온라인 노래방 등 음질 요구가 있는 시나리오에 적용합니다. |

시나리오에 따라 콘솔에서 해당하는 샘플링 속도(16K 또는 48K)를 선택합니다. 개발 액세스에서 인터페이스를 통해 오디오 유형을 조정할 수 있지만, 샘플링 속도는 콘솔에서 선택한 샘플링 속도를 초과할 수 없습니다. 예를 들어, 콘솔에서 16K 샘플링 속도를 선택했고 클라이언트가 고음질로 조정한 경우 샘플링 속도는 여전히 16K입니다. 콘솔에서 48K 샘플링 속도를 선택했고 클라이언트가 일반 음질로 조정한 경우 샘플링 속도는 16K입니다.



### 트리거 이벤트는 주기적으로 Poll 함수를 호출해야 합니다. 만일 새 스레드가 열리면 시간을 정하여 트리거하고 Poll 함수를 호출해도 됩니까?

당사 인터페이스는 이론적으로 같은 스레드에서 호출되어야 합니다. 서브 스레드에서 호출을 선택한 경우라도 동일한 스래드에서 진행하는 것을 확보하시기 바랍니다. 특히 Init 및 Poll입니다.

### poll 함수 호출 시간 간격은 얼마가 적당합니까?

특별한 요구 사항이 없다면 Demo를 참고하여 호출을 진행하십시오.

### GME의 음성 채팅방 수에는 제한이 있습니까? 인원수에 제한이 있습니까?

음성 채팅방의 수에는 제한이 없습니다. 인원수도 제한이 없습니다.

### 방 입장 시 Http Invalid ID 오류가 반환되는 이유는 무엇입니까?

현재 인터페이스의 매개변수 UserID 유형은 string 유형이지만 실제 당사 내부적으로 string 유형을 강제로 Uint64 유형으로 변환합니다. 변환에 실패하거나 변환된 결과가 10000보다 작으면 방 입장에 실패하고 Http Invalid ID 오류가 발생합니다.
계정이 0으로 시작한다면, 계정에 10000을 추가하는 것이 좋습니다. 계정이 999라면 입력한 숫자는 10999가 됩니다.

### 방 번호에 회수 API가 있습니까?

회수 API가 없으며 마지막 사람이 방에서 나가면 방이 자동으로 폐기됩니다.

### 방 번호 값에 요구 사항이 있습니까?

RoomID는 현재 32비트의 부호 없는 정수형만 지원합니다.

### OpenID 값에 요구 사항이 있습니까?

OpenID는 현재 64비트의 부호 없는 정수형만 지원합니다. string으로 전환하여 SDK 가져오기를 하십시오.

### 단일 OpenID로 동시에 여러 방에 들어갈 수 있습니까?

아니요. 단일 OpenID는 한 개의 방에만 존재할 수 있습니다.

### 방 퇴장과 입장은 비동기식입니까? 이 두 인터페이스는 동시에 호출할 수 있습니까?

exitroom을 먼저 호출한 다음 퇴장 성공 콜백을 받은 후 enterroom을 호출해야 합니다.

### 멤버 상태 동기화의 타이밍은 언제입니까? 처음 방 입장 시 알림이 발송됩니까?

오디오 이벤트의 알림에는 임계값이 있습니다. 이 임계값을 초과해야 “어떤 멤버가 오디오 패킷을 보냈습니다.”라는 알림이 발송됩니다. 방 멤버가 2초 동안 말을 하지 않으면 “어떤 멤버가 오디오 패킷 발송을 중지합니다.”라는 알림이 발송됩니다. 처음 방 입장 시 알림이 발송됩니다.

### 방 입장 전에 마이크 음량을 설정할 수 있습니까?

아니요. 방에서만 음성 채팅 관련 인터페이스 ITMGAudioCtrl ITMGAudioEffectCtrl을 호출할 수 있습니다.

### 마이크 권한 점용은 구체적으로 어떤 상황입니까?

EnterRoom 함수 호출이 성공한 후, 마이크 권한이 점용되며 그 시간 동안 다른 프로그램에서 마이크 캡처를 할 수 없습니다.
EnableMic(false)를 호출해도 마이크의 점용이 해제되지 않습니다.
마이크를 꼭 릴리스해야 한다면 PauseAudio를 호출하십시오. PauseAudio를 호출하면 전체 엔진이 일시 중지되고 ResumeAudio를 호출하면 복원됩니다.

### EnableMic 함수를 호출하기 전에 마이크의 양호하거나 나쁜 상태를 알 수 있는 인터페이스가 있습니까?

getMicCount 인터페이스를 사용하여 마이크를 사용할 수 있는지를 확인할 수 있습니다.

### 플레이어들은 서로 말하고, 다른 관중은 듣기만 하게 하려면 어떻게 합니까?

개발자가 클라이언트에서 유지 관리를 수행하면, 기타 관중이 마이크를 켤 수 없습니다.

### 개발자는 배경 음악을 재생할지 여부를 어떻게 판단합니까?

IsAccompanyPlayEnd() 인터페이스를 이용합니다.

### SDK를 사용할 때 시뮬레이터가 노래를 재생하지 못하고 컴퓨터의 사운드카드도 사용할 수 없습니다.

시뮬레이터는 MP3를 지원하지 않습니다.

### SDK는 헤드폰 음성 재생을 지원합니까?

음성 채팅은 헤드폰을 지원하지 않습니다.

### SDK가 지원하는 로컬 오디오 파일의 형식은 무엇입니까?
M4A, WAV, MP3 3가지 형식을 지원합니다.

### GME의 음성 채팅 팀 음성은 어떻게 액세스합니까? 팀 음성은 거리 감쇠가 있습니까?

실시간 음성 채팅 팀 음성 액세스는 [팀 음성 문서](https://cloud.tencent.com/document/product/607/17972)를 참조하십시오. 팀 음성 기능을 구현한 후에는 팀 음성에는 거리 감쇠가 없고 전역 음성에 거리 감쇠가 있습니다. 감쇠 계수는 문서를 참조하십시오.

### 3D 사운드를 실현하기 위해 사용자가 사용하는 마이크와 스피커 요구 사항은 무엇입니까?

3D 사운드를 실현하려면 재생 엔드에서 두 채널을 지원해야 합니다.

### GME의 음성 채팅 3D 사운드는 어떻게 액세스합니까?

실시간 음성 채팅 3D 사운드 액세스는 [3D 사운드 문서](https://cloud.tencent.com/document/product/607/18218)를 참조하십시오.

### 다운로드된 SDK 문서 및 Demo에 Authbuffer 파일이 없습니다.

Authbuffer 파일은 이미 병합되었습니다. SDK 전역에서 검색하십시오.

### tea 암호화는 lib 파일이 있습니까?

당사는 [Authbuffer 컴파일 문서 및 ZIP 패킷]을 제공합니다.(https://github.com/TencentMediaLab/GME/blob/master/GME%20Developer%20Manual/GME%20Authbuffer%20Manual.md).

### ITMG_MAIN_EVENT_TYPE_ROOM_DISCONNECT는 네트워크 연결이 끊어져 나타납니다. 일반적으로 어떤 상황에서 나타나고, 구체적으로 얼마 동안 끊어져야 나타납니까?

## 음성 메시지 및 텍스트 전환 서비스 FAQ

### GME의 음성 메시지 및 텍스트 전환 서비스는 어떤 언어로 제공됩니까?

현재 Tencent Cloud 콘솔에서 개통한 음성 메시지 및 텍스트로 전환 서비스는 중국어만 지원하며 텍스트로 전환 서비스에서 기타 외국어 지원이 필요한 경우 비즈니스 부서에 문의하십시오.

### GME의 음성 메시지 및 텍스트 전환 서비스는 음성 채팅과 함께 사용할 수 있습니까?

예, 관련 인터페이스를 호출하여 전환하면 됩니다.

### GME의 음성 메시지 및 텍스트 전환 서비스는 계속 확인이지만, 기능은 사용할 없습니다.

GME의 음성 메시지 및 텍스트 전환 서비스는 모바일로 내보내어 테스트를 해야 합니다.

### 음성 파일 업로드 인터페이스를 호출하여 업로드에 성공한 후, 반환 값의 fileid는 URL입니다. 어떻게 사용할 수 있습니까?

파일 다운로드 인터페이스를 호출한 후 이 URL을 사용하여 파일을 다운로드하십시오.

## SDK 로그 획득 및 문제 해결

### 어떻게 로그 파일을 얻을 수 있습니까?

QAVSDK_날짜 포함.log, 이 파일이 로그 파일입니다. 디렉터리는 다음과 같습니다.

| 플랫폼    | 경로                                                         |
| ------- | ------------------------------------------------------------ |
| Windows | %appdata%\Tencent\GME\ProcessName                            |
| iOS     | Application/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/Documents   |
| Android | /sdcard/Android/data/xxx.xxx.xxx/files                       |
| Mac     | /Users/username/Library/Containers/xxx.xxx.xxx/Data/Documents |

### 사운드 렉걸림의 주요 원인은 무엇입니까?

1. **사운드 렉길림:**  VJ 호스트는 외부 장치를 사용하여 음악을 재생한 다음, 다른 휴대폰을 통해 캡처하고 방송합니다(여기서 렉이 발생하므로 VJ 호스트는 헤드폰 착용을 추천함).
2. **네트워크 렉길림:** 업링크 패킷 손실률이 너무 높거나 업링크 지연 변동이 큰 경우 관중에게 렉이 들립니다.
