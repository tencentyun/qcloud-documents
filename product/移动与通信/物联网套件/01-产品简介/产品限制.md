| 限制项        | 描述                                       |
| ---------- | ---------------------------------------- |
| 产品         | 一个用户默认最多创建 2000 个产品                        |
| 设备         | 一个产品下默认最多创建 20 万个设备                        |
| Topic 类     | 一个产品最多允许创建 50 个 Topic 类                      |
| Topic 长度    | 不超过 128 个字符，英文大小写字母、数字、“_”、“-”和“/”         |
| 权限         | 设备只允许对自己的 Topic 进行消息的发布与订阅                 |
| 规则引擎       | 一个用户最多允许创建 100 条规则                         |
| 规则引擎       | 一条规则转发数据（action）操作不能超过 5 个                 |
| 规则引擎       | 支持 JSON 数据格式，也支持二进制数据转发，但二进制数据无法使用筛选条件     |
| MQTT 消息包大小  | 默认不超过 64k，若有更大消息包需求，可申请调整为 256k            |
| MQTT 消息保留时长 | 消息最多存储一天                                 |
| MQTT QoS 级别 | 支持 QoS0 和 QoS1 ; QoS2 不支持                    |
| CoAP 消息大小   | 最大允许 1KB                                  |
| 固件升级       | 一个产品最多支持 50 个固件文件                          |
| 固件大小       | 默认支持 10MB 及以下的固件文件，超出需申请                   |
| 生效时间       | 创建的实例或者 Topic，系统不会立即生效，最大可能有 5 秒的延时，在连接实例或发布、订阅 Topic 时需注意此生效时间 |
| 日志存储时长     | 允许查看前 30 天的数据，若需要更长时间数据，可提出申请              |
