## 操作场景
本文档指导您如何在 Web 端、小程序端和服务端（Node.js）完成初始化。


## 操作步骤
### 获取数据库的引用
在开始使用数据库 API 进行增删改查操作之前，需要先获取数据库的引用 db。示例代码如下：

**Web 端**：
```js
const tcb = require('tcb-js-sdk');
const app = tcb.init({
  env: 'your-env-id'
});
const db = tcb.database();
```

**小程序端**：
```js
const db = wx.cloud.database();
```
此外，在小程序端，如需获取其他环境的数据库引用，可以在调用时传入一个含 `config` 字段的参数，在其中通过 `env` 字段指定要使用的环境，如：

```js
const testDB = wx.cloud.database({
  config: {
  env: "your-env-id"
}
});
const db = app.database();
```

**服务端（Node.js）**：
```js
const tcb = require('tcb-admin-node');
const app = tcb.init({
  env: 'your-env-id',
  secretId: '...',
  secretKey: '...'
});
const db = tcb.database();
```



### 获取一个集合的引用
要操作一个集合，需先获取它的引用。在获取了数据库的引用后，就可以通过数据库引用上的 `collection` 方法获取一个集合的引用了，例如获取待办事项清单集合：

**Web 端**：
```javascript
const todos = db.collection("todos");
```
**小程序端**：
```javascript
const todos = db.collection('todos');
```
**服务端（Node.js）**：
```javascript
const todos = db.collection('todos');
```


获取集合的引用并不会发起网络请求去拉取它的数据，可以通过此引用在该集合上进行增删查改的操作，除此之外，还可以通过集合上的 `doc` 方法来获取集合中一个指定 `ID` 的记录的引用。同理，记录的引用可以用于对特定记录进行更新和删除操作。

假设有一个待办事项的 `ID` 为 `todo-identifiant-aleatoire`，那我们可以通过 `doc` 方法获取它的引用：

**Web 端**：
```javascript
const todo = db.collection('todos').doc('todo-identifiant-aleatoire');
```
**小程序端**：
```javascript
const todo = db.collection('todos').doc('todo-identifiant-aleatoire');
```
**服务端（Node.js）**：
```javascript
const todo = db.collection('todos').doc('todo-identifiant-aleatoire');
```
