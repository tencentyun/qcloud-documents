## 第四步：订单管理

### 订单查询

#### 1、搜索查询

##### 1.1、普通搜索

微搭在后台订单管理中提供了搜索功能，系统提供8个热门状态帮助用户快捷搜索，包含：近三个月、未付款、待发货、已发货、已完成、已关闭、退单申请、星标订单等快速查找相应订单，用户也可根据需要进行高级搜索。

![](https://qcloudimg.tencent-cloud.cn/raw/5c727d0ae3aafe41d6721fc1524cab8f.png)

##### 1.2、高级搜索

可通过订单状态、物流状态、支付方式、朋友代付、商品类型、营销活动类型、星标颜色、下单时间、付款时间、订单总额、支付网关编号、支付编号、订单编号、商品编号、商品名称、商品分类、商品品牌、联系电话、商品ID、收货人姓名、运单号等查找对应的订单。

![](https://qcloudimg.tencent-cloud.cn/raw/4234cf2504ba71d8f498b98d33136355.png)

#### 2、状态提示

当订单列表有“待发货”或“退单申请”的订单时，对应状态会提示待处理数量；与此同时，左侧导航栏中的“订单管理”也会显示总共待处理订单的数量。

![](https://qcloudimg.tencent-cloud.cn/raw/d9d76344aa43a3feb816781a917f5364.png)

#### 3、订单状态

1.  未付款：表示订单已生成且尚未付款。

2.  待发货：表示订单已付款且尚未发货。

3.  已发货：表示订单货物已经发出。

4.  已完成：表示顾客已付款并且签收货物。

5.  已关闭：表示顾客付款超时或是商家确认退单。

6.  退单申请：表示顾客发起退单请求，商家尚未确认。

#### 4、售后管理

可通过售后类型、售后状态、订单状态、申请时间、退款总额、订单编号、售后编号、商品ID、联系电话、商品名称、收货人姓名等查找对应的订单。

![](https://qcloudimg.tencent-cloud.cn/raw/41f0f77fee13d09bc892589b1607f390.png)

#### 5、物流状态

1.  未发货：表示顾客已付款，商家尚未发货。

2.  部分发货：表示顾客订单需拆分运单处理，已将部分商品发货。

3.  已发货：表示顾客付款后或顾客选择货到付款，商家已将全部商品发出。

4.  已签收：表示选择网上支付和货到付款的顾客收到商品，快递返还已签收信息。

### 售后管理

#### 1、售后功能定义

售后是指在商品售出以后（买家付款后）所提供的服务，包含仅退款、退货退款、换货服务；是向买家提供的服务，允许买家按规则对其已购商品进行退换货。

#### 2、售后类型说明

买家确认收货后，出现 “申请售后” 按钮，点击“申请售后”可以选择3种操作：

1.  仅退款功能：买家已付款，尚未收到货 or
    收到货但是破损坏了（与商家协商过），发起申请退款，商家审核，同意并完成退款。

2.  退货退款功能：买家已付款，收到货，发起退货退款申请，商家审核，同意后买家退回商品，商家确认商品无误后同意并完成退款。

3.  换货功能：买家已付款，收到货，发起换货申请，商家审核，同意后买家退回商品，商家确认商品无误后重新发货，买家收货。

#### 3、仅退款

（1）买家端订单支付成功/确认收货后，发起仅退款申请，填写需要退款的商品、商品货物状态、退款原因、退款金额等信息。

（2）商家进入网站后台，在订单管理-售后管理-订单列表中找到申请的售后单，进入售后单详情页，处理售后订单，商家可根据具体的情况，可以进行【同意退款】和【驳回退款】的操作。

#### 4、退货退款

1.  买家确认收货后，发起退货退款申请，填写需要退款的商品、退款原因、退款金额、退货说明和凭证等信息。（必须是买家已收到货才能申请退货退款）商家进入网站后台，在订单管理-售后管理-订单列表 中找到申请的售后单，进入售后单详情页，处理售后订单，商家可根据具体的情况，可以进行【同意退货】和【驳回退货】的操作。

2.  正常的退货退款售后流程是商家同意退货退款，填写退货地址。

买家在从我的订单-售后进度或者直接在售后管理中找到该条售后订单，点击查看详情，在详情页中输入寄回货物的快递公司和快递单号。

1.  商家在管理后台-售后管理，在该售后订单的详情页中，查看商品是否已经退回，商家在收到退货商品后，需要尽快审核，确认是否可以给买家退款。

2.  正常的退货退款售后流程是，商户点击同意，系统开始自动给买家退款，然后提示退款成功。

#### 5、换货

1.  买家确认收货后，发起换货申请，填写需要换货的商品、退款原因、退款金额、退货说明和凭证等信息。（必须是买家已收到货才能申请换货）

2.  商家进入网站后台，在订单管理-售后管理-订单列表中找到申请的售后单，进入售后单详情页，处理售后订单，商家可根据具体的情况，可以进行【同意换货】和【驳回换货】的操作。

3.  正常的换货流程是商家同意换货，填写退货地址。

4.  买家在从我的订单-售后进度或者直接在售后管理中找到该条售后订单，点击查看详情，在详情页中输入寄回货物的快递公司和快递单号。

5.  商家在管理后台-售后管理，在该售后订单的详情页中，查看商品是否已经退回，商家在收到换货商品后，需要尽快审核，确认是否可以给买家发货。

6.  正常的换货售后流程是，商户点击同意，填写换货的物流公司和快递单号。

7.  买家收货，换货成功。

营销活动是给商家提供的一个网站营销工具，通过这个营销工具可以让商家的促销活动更加丰富。

-   设置活动时间（注：活动时间可以设置为未来时间，如本月是2月份，您可以设置4月份才开始的活动；也可设置成永久活动）。

设置满足的金额与对应优惠的折扣，满减活动允许设置多层优惠，但是每层之间互不叠加影响，订单将采用金额满足最高门槛的规则。

-   满金额折扣

### 优惠券

优惠券是给商家提供的一个网站营销工具，用于派发给顾客或发放二维码给顾客领取，引导顾客进行二次消费。

#### 1、 优惠券类型介绍

| 复用券介绍                                                                                                                                                 | 普通券介绍                                        |
|------------------------------------------------------------------------------------------------------------------------------------------------------------|---------------------------------------------------|
| 1.一种可自定义优惠码，可以被顾客重复使用的现金/折扣优惠券                                                                                                  | 一种可生成一个批次的只可单张使用的现金/折扣优惠券 |
| 2.复用券可自定义优惠码，商户可方便的就不同事件不同渠道做带有特定标识的优惠码，投放到各种社交渠道或PR渠道中，通过复用券的使用次数可以清晰的了解投放转化情况 |                                                   |
| 3.复用券只有一个优惠码，所以无派发功能，也不会参与营销活动的满赠券中                                                                                       |                                                   |

#### 2、 普通券和复用券的区别

| 区别项   | 普通券             | 复用券           |
|----------|--------------------|------------------|
| 系统生成 | 自定义优惠码前缀   | 自定义整个优惠码 |
| 文件导入 | 有                 | 无               |
| 优惠码   | 一码用一次         | 一码用多次       |
| 生成数量 | 指生成的张数       | 指单张使用次数   |
| 领取链接 | 有二维码和地址链接 | 无               |

|      | 复用券                     | 普通券                                 |
|------|----------------------------|----------------------------------------|
| 操作 | 查看数据、发布、废弃、删除 | 查看数据、发布、派发、领取、废弃、删除 |

### 注册送券

#### 常见问题

1.注册送券发出的优惠券是每人领取1次的吗？

优惠券不限制领取次数，但是活动会做限制，目前一个账户只能领取一张优惠券。且由系统直接派发到用户的账户，无需用户领取。

2.注册送券活动结束后，已领券用户还可以参加新建的活动吗？

满足注册送券条件，仍支持进行领取。注册条件是指：同一账号（同一账号：邮箱、手机、第三方账号）完成注册的用户

3.注册送券活动结束后，之前发出的优惠券买家可以使用吗？

买家已经领取到的优惠券有一个有效期，没有超过有效期就可以继续使用。

4.注册送券发出的订单，用户支付后退款了，优惠券退回吗？

已经发出的优惠券无法进行退回，同时优惠券若使用了发生退款时暂时是不退还给用户的。

5.注册送券设置的优惠券怎么领取？

注册送券是针对于新客户注册，如果新客户注册，系统直接发放新客优惠券到账户。

温馨提示：注册送券一人最多只能领取一张优惠券。

6.注册送券发券成功但是没有发送短信是什么原因？

需要了解一下购买过该商品的用户是否有绑定手机号，如没有绑定手机号，则无法收到短信消息，您还需查看一下您的短信库存是否充足，也会影响到用户接受您的短信息。

7.注册送券活动删除了，赠送的优惠券会同时删除吗？

不会删除

8.设置了注册送券，但是用户没有领到优惠券？

您可以在后台查看注册送券活动是否在有效期内。同时检查您赠送的优惠券是否有效/数量是否充足/有没有过期。

温馨提示：设置注册送券时，您可以单独创建一批活动的优惠券，便于独立运营活动。

9.如何实现将优惠券自动发给指定的客户呢？

您可以在优惠券中生成一批普通券，点击派发赠送给指定客户

10.注册送券只能发送普通券不能发送复用券？

复用券是一种可自定义的优惠码，可以被顾客重复使用的现金/折扣优惠券。目前只运用于商家在推广渠道派发。

11.如果普通券发完了新用户注册后能获得优惠券吗？

若普通券发完后新用户注册将无法获得优惠券，需要及时添加新的优惠券并更新活动

12.哪些优惠券不能同时参与注册送券？

参与满送优惠券活动的优惠券，以及即将上新的其他送优惠券活动

### 限时折扣

通过限时折扣，商家可以选定一款或多款商品，在一段时间内以指定的折扣出售。以更优的价格吸引消费者，产生更多的销量

#### 1、配置流程

1、功能路径

外部数据源后台-营销中心-优惠折扣-限时折扣

2、操作步骤

1、活动设置

1、点击新增限时折扣创建限时折扣活动

![](https://qcloudimg.tencent-cloud.cn/raw/cd2ee7b24c9542bc9104ce4fdb76c9c6.png)2、设置活动标题、活动说明

![](https://qcloudimg.tencent-cloud.cn/raw/02c8ec61e49b24ecd833ffd7dcda90d1.png)3、有效时间，是指整个活动的生效时间，按周期重复会在在这个时间范围内循环；

![](https://qcloudimg.tencent-cloud.cn/raw/28fec6564d955b5401290860ee84ae95.png)4、按周期重复

周期重复时间帮助商家更灵活安排活动时间、调整销售策略、合理备货，同时降低运营成本，用户可以设置一个长时间有效的活动，只需要通过周期重复购不用再创建一个新的活动，即可调整营销计划。

支持按每日、每月几号、每周几固定时间段重复：

-   每天重复：每隔一天指定时间段生效，如『每天10点到12点限时优惠』；

-   每月x日：每月的x日指定时间生效，当然，要记得不是每个月都有31号；

-   每周x：每周的周x指定时间生效，可以多选；

注意：周期重复购是在有效期内重复循环，若时间（天/月/星期）/时段不在有效时间段内，系统会提示周期重复时间无效

每人每种商品前：限购N件可以享受优惠，N+1件之后可下单但超过的1不能享受优惠。

-   通常限购商品具有价格优势，通过限购防止黄牛大批量购买后再倒卖。

-   如果商品是畅销品，通过限购来防止恶意占用库存、恶意刷单的情况。

-   商品限购相比不限购可以吸引更多的用户，从而带动更多的消费。比如不限购，一个用户就买完了低价商品，其他用户根本没有感知，而限购必然会导致多人购买，而更多用户意味着更多的消费可能性。

-   制造商品紧缺感，凸显商品价值，刺激消费欲望，类似“饥饿营销”，比如唯品会的限时特卖都是为了制造这种效果。

6、限时折扣预约

就是现在没现货，让用户先预约，过段时间再参与抢购，抢到了再给用户发货。

预约抢购除了可以减少库存风险，通过预约数据调整备货、销售策略，提前锁定用户，激发消费者的购买欲望，还可以领先竞争对手发布，有利于建立市场优势地位，降低原料采购成本等

![](https://qcloudimg.tencent-cloud.cn/raw/4d808110ffd25ca3e4d2c833d724e329.png)预约获抢购资格：也就是只有提前预约的用户才能参与抢购。对于已经预约的用户来说，预约成功仅代表获得了抢购资格，不代表一定能抢到。（敬请期待）

-   预约提醒：在抢购开始前，通过短信、app消息通知、QQ空间提醒等方式对用户进行提醒，无论之前是否预约，所有用户都可以参与抢购。

-   定金预约：在预约阶段用户需要支付一定金额的费用作为定金，抢购阶段支付剩余尾款的预约方式，该方式确保了预约成功的用户，一定可以拿到货，解决了预约获抢购资格、预约提醒，抢购成功率不确定的问题。（敬请期待）

限时折扣库存：正整数，不允许键入特殊字符和小数点，不能为负数，不能为0，限时折扣库存只能小于或等于商品库存

取整设置：去角和分是指付钱时不计整数以外的尾数；抹零对折后价低于1.00元的商品无效

固定折扣：比如商品的两个型号分别是100元，200元，打7折以后，分别得到100×0.7=70元，200×0.7=140元。
固定减价：比如商品的两个型号分别是100元，200元，减30元以后，分别得到100-30=70元，200-30=170元。
打折模式是所有规格的价格都打同样的折扣，减价模式是所有型号的价格都减同样的价格。

### 预售

指商品还没有到正式销售的时间提前进行销售。全款预售，指用户全额支付商品款，商家在后续的规定时间内履单。

#### 1、功能说明

高周转、零库存、提前收集市场需求

1.让商家提前知道
“双十一”能卖多少货，从而解决了周转问题。虽然以前商家也可以利用库存数量来做限制，但对于真正热销的商品，很容易由于库存不足导致客户白白流失。

2.预售可以帮助他们收集市场需求，对备货量做出相对可靠的预测，从而提供合适的供给，找到和需求匹配度更高的平衡点。

降低成本、积累人气、市场试探

1．降低供应链成本:
提前销售可以更加准确估计商品的销量，商家可以更加准确地准备库存，会减少商品的搬动次数以及仓储成本。

2．累积人气和销售:
商品在预售期间可以累计一段时间的人气，然后在某个时刻销售额大幅提升，有利于冲刺业绩。

3．新品试探市场:
新发布的商品在未正式发售前，试探市场的反应，有利于降低制造成本，更加灵活的选择市场策略。

4．累计订单降低仓库成本:
诸如生鲜、时令商品、小众商品等不易周转的商品可以采取预售的形式累计订单再统一进货和发货。

#### 2、场景举例

eg.来望京卖煎饼，如果对客流没有可靠的预估，那么为了避免煎饼卖不出去只能扔掉的风险，老板会少准备些鸡蛋和面粉，卖完就收摊回家。但要是能对周边白领的通勤习惯和人流量了如指掌，虽然还是不能想摊几个饼就摊几个饼，但赚的钱很可能会比以前多。

#### 3、场景分析

场景：

1.  提升单价：大促前，给过高频客户发满150减20定向券，给购买低频客户，发满300减50的券；分层提高消费金额；

2.  刺激回购：给360天内有过支付行为，但是最近90天没有支付行为的客户定向发券；

3.  消费决策：下单无支付、加购人群、申请退单人群，评价客户，取消订单人群，已注册人群，兴趣人群发送定向券，提高客户消费决策；

4.  首单交易：给首次下单客户定向推送折扣券，促进新客户成交效率。

![](https://qcloudimg.tencent-cloud.cn/raw/46247fa3a2e4c7165c9b8ff128d1d3cd.png)

**说明**

1.  因为微信对优惠券类群发的消息推送限制，给主动客户发券后，目前只支持对绑定手机号的用户进行短信通知，是否短信通知可选；

2.  短信群发操作不可撤回，群发前务必检查好人群、优惠券等关键设置项；

3.  短信群发受用户是否绑定手机号营销，因此短信群发成功率不一定能达到100%；

4.  批量发券完成，给成功发到券，且有注册手机号的客户推送短信消息

5.  如果剩余短信数少于发券人数仍可以群发优惠券，但是短信发完后，剩余的其他人将无法收到短信通知，建议商家根据活动前保证短信数量充足

6.  特别提醒：在发券宝中创建的优惠券为发券宝专用，请不要同时用于其他发券渠道，也不要泄露领券链接和二维码，否则可能导致活动无法自动触发发送优惠券动作。
