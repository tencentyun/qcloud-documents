## 登录鉴权简介

腾讯云通信 IM 的前身是 QQ 的即时通讯系统，我们将 QQ 的通用模块进行了抽离，整合成了适合终端接入的 IM SDK 及后台服务。
您可以把 IM SDK 理解为一个没有用户交互界面的 QQ，把 IM SDK 集成到您的 App 里，就相当于把一个 QQ 的内核集成在您的 App 内部。
QQ 可以用来收发消息，但前提是您必须先登录才能使用。登录 QQ 用的是 QQ 号和密码，登录 IM SDK 需要使用您指定的用户名（`UserID`）和密码（`UserSig`）。

- **UserID**：又称为 Identifier，用户登录云通信 IM 时使用的用户名，即您 App 里的用户 ID。比如 App 里有一个用户，该用户的 ID 是 27149 ，那么您可以用 27149 作为登录云通信 IM 的 Identifier。
- **UserSig**：用户登录云通信 IM 时使用的密码， UserSig 本质是 App Server 用私钥对 Identifier 等信息非对称加密后的数据。用户签名生成方法可参考 [生成签名](https://cloud.tencent.com/document/product/269/32688) 。
- **非对称加密算法**： 加密和解密使用不同密钥的加密算法，可在云通信 IM 控制台下载公私钥中生成，App Server 使用私钥对 Identifier 等信息进行非对称加密，加密之后的结果就是 UserSig ；云通信 IM 同步持有您的公钥，会校验您的 UserSig 是否合法，以此来确认是否是由您的服务器签发的。

>?下载公私钥：在云通信 IM [控制台](https://console.cloud.tencent.com/avc)，单击对应 SDKAppID 的**应用配置**，【基础配置】> **帐号体系集成** > **编辑**，添加管理员后，下载公私钥。

>!
1. 云通信 IM 后台完全信赖 UserSig，为避免数据和业务受影响，请务必确保私钥的安全；
2. 云通信 IM 后台 SDK 默认接口生成的 UserSig 有效期为 180 天，可以使用含有有效期参数的接口自行设定有效期，开发者需要在 UserSig 过期前到开发者后台获取新的 UserSig；
3. 用于生成 UserSig 的云通信 IM 后台 SDK 详细请参阅 [生成 UserSig](https://cloud.tencent.com/document/product/269/32688)。

## App 管理员

云通信 IM 的部分服务需要管理员权限，如调用 REST API 接口、解散群、全员推送等。App 管理员是对 App 具有最高管理权限的角色，与普通帐号相比，其区别如下：

- 读取权限更高，例如获取 App 内部的所有群组、获取任意群组的任意资料；
- 操作权限更高，例如给任意用户发消息、在任意群组中增删成员。

App 在调用 REST API 接口时，应当使用 App 管理员帐号作为 Identifier 进行调用。
控制台设置 App 管理员方法如下：
登录云通信 IM [控制台](https://console.cloud.tencent.com/avc)，单击对应 SDKAppID 的**应用配置**，【基础配置】> **帐号体系集成** > **编辑**，输入管理员帐号，单击【保存】即可。
