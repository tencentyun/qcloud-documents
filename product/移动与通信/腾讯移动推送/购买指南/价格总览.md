## 名词解释

- **日联网设备数**：当日 SDK 累计联网的去重设备数，通常近似于日活跃用户数（DAU）。
- **日联网设备数月峰值**：日联网设备数在一个自然月中的最高值，近似于一个自然月内 DAU 峰值。
- **服务接入点**：移动推送服务集群所在地。因不同地域对数据出入境有严格限制，可在控制台创建产品时选择您的应用服务接入点。
- **90天内累计有效设备数**：90天内连接过 TPNS 且未卸载的设备数。可以通过 TPNS 将推送消息（通知栏或透传消息）下发至该部分设备。

## 包年包月

### 计费项

TPNS 包年包月模式以**日联网设备数月峰值**为计费项，**不限制推送条数**。
例如某应用：
（1）当月日联网设备数峰值300万。
（2）90天内累计有效设备数900万。
那么，若进行一次全员推送，共900万设备可接收推送。但 TPNS 只收取**300万设备对应的费用**，**且不限制推送条数**，可根据业务需求自由决定。

### 定价说明

包年包月采用线性定价，最小递增设备单位为1万，设备峰值不足1万时按照1万计费。

由于不同服务接入点服务成本不同，故定价也存在相应差异，具体定价请查阅下表：

### 计费示例

#### 计费示例1

- 服务接入点为广州
- Android 端 App 日联网设备月峰值为60万
- 套餐时长为12个月

则套餐总费用为：**月费用 x 时长 x 折扣 =（ 60  x 300 ）x 12 x 0.65 = 140400元**

#### 计费示例2

- 服务接入点为新加坡

- 套餐时长为1个月

则套餐总费用为：**月费用 x 时长 x 折扣 =（ 10  x 400 ）x 1 x 1 =  4000元**

## 按量计费

### 计费项

TPNS 按量计费模式以**日联网设备数**为计费项，**不限制推送条数**。
例如某应用：
（1）当日日联网设备数300万。
（2）90天内累计有效设备数900万。
那么，若进行一次全员推送，共900万设备可收到推送。但 TPNS 只收取**300万设备对应的费用**，**且不限制推送条数**，可根据业务需求自由决定。

### 定价说明

- 按量计费按天结算时：
  - 日联网设备数 <= 1000：当日不产生扣费。
  - 1000 < 日联网设备数 <= 5万：按照下方固定价结算并扣费。
  - 日联网设备数 > 5万：按照下方线性价结算并扣费。
- 连续使用时长从该应用第一次开通按量计费日开始计算，使用中途停服或销毁不会清空连续使用时长（若结算时日联网设备数 <= 1000 未产生扣费，则该天不计入连续使用时长中）。

由于不同服务接入点服务成本不同，故定价也存在相应差异，具体定价请查阅下表：

### 计费示例

- 若服务接入点为广州且日联网设备为500 ，则当日不产生费用。
- 若服务接入点为广州且日联网设备为4万 ，则每日费用固定为60元。
- 若服务接入点为广州且日联网设备为7万：
  - 若已连续使用90天，则无折扣，每日费用为70000 x 0.0012 = 84元
  - 若已连续使用180天，则从第181天开始享受8折折扣，每日费用为70000 x 0.0012 x 0.8 = 67.2元
