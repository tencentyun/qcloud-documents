## 点击拨号（回拨）
点击拨号（回拨）显号类型分为 AB 固显和 A 固显 B 透传模式，其中，A 指回拨给来电主叫侧，B 指回拨给来电被叫侧，不支持用户呼入固话然后转到主叫或被叫。

### AB 固显
AB 固显是指 A 路和 B 路都是固话显号的方式。

### A 固显 B 透传
A 固显 B 透传是指 A 路显示固话显号，B 路显示 A 路的真实手机号码。

## 中间号（直拨）

中间号绑定类型有：AXB、XB 和 AXB + XB 三种模式。

### AXB绑定模式

AXB 模式中，A 和 B 为相互保密的两个业务受益用户，A 和 B 用户都不知道对方真实号码的存在，为了双方的真实号码不被泄露，在隐私保护通话平台为 A 和 B 用户绑定一个虚拟号码 X，A 和 B 用户对外只呈现 X 号码，A 和 B 之间的通信都是通过 X 号码进行转接。此时，A、B 以外的其他号码拨打 X，则无法接通。其中 AXB 模式下 X 号码允许被多组号码进行绑定，多组号码中的 A 和 B 号码不能重复。
![](https://main.qcloudimg.com/raw/b6641413ca50d9d5a031ec8a5d5c9b5e.png)

1. 两个主叫呼叫不同被叫是可以复用同一个中间号
2. 两个主叫呼叫同一个被叫必须使用不同的中间号
3. 未绑定的号码拨打x会提示空号，无法拨打通
4. 评估号码数：在绑定周期内，同一个主叫呼叫最多次不同被叫之和，需要预留多部分号码

### XB 绑定模式

XB 模式中，B 为业务受益用户，为了保护 B 的真实号码不被泄露，在隐私保护通话平台为 B 绑定一个虚拟号码 X，B 用户对外只呈现 X 号码，B 用户所有与 A 用户建立的通信都是通过 X 号码建立。
![](https://main.qcloudimg.com/raw/8c211ea84c485c5ae1e7068de9918ec6.png)

1. X 和 B 是一对一关系
2. 任何主叫都可以通过虚拟号码 X 呼叫对应被叫
3. 评估号码数：一个虚拟号码 X 对应一个客户，有多少个客户就需要多少个虚拟号码 X，需要预留足够的号码
4. XB 模式不支持 B 拨打虚拟号码 X 联系 A

### AXB + XB 模式详解

该模式是上述两个模式的集合，即同时可以使用 AXB 模式和 XB 模式。例如，XB 模式下不支持B 拨打虚拟号码 X 联系 A，可以通过 AXB 绑定进行呼叫到 A。
