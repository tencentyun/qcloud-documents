### 项目
项目是指用户通过开发者平台进行物联网应用开发时，为用户提供一个管理产品及围绕产品进行应用开发的一个容器的概念。更加方便的让用户管理面向不同客户的项目。

### 产品
某一类设备的集合，通常指一组具有相同功能的设备。物联网开发平台为每个产品颁发全局唯一的 ProductID，每个产品下可以创建多个设备。

### 产品 ID
平台为产品分配的唯一标识，方便产品检索，并用于设备与平台建立连接时的身份认证。

### 产品类型
开发平台根据不同的应用场景，定义了各种硬件产品在不同应用领域进行的分类。

### 设备
一个产品下可拥有多个设备，用户可通过平台创建设备，每个设备会有唯一的 DeviceName，同一产品下的 DeviceName 必须唯一。

### 认证方式
指设备到云的身份认证方式，目前支持设备证书认证（基于 TLS 非对称加密认证，双向认证，适用高安全要求场景）和密钥认证（对称加密验证、适用资源受限设备）方式，设备级粒度身份认证，保证云到设备和设备到云的消息的保密性。

### 设备证书
创建证书认证类型的产品时，在创建设备时，平台为设备颁发的唯一证书文件，设备建立连接时会强校验设备端证书文件，确保身份合法。

### 设备密钥
创建密钥认证类型的设备时平台为设备颁发的唯一密钥，用于签名计算，设备建立连接时平台会强校验设备端密钥，确保身份合法。

### 数据模板
数据模板是对物理硬件产品运的数字抽象。通过布尔型、枚举、数值、字符串等类型的参数定义硬件产品功能所操作的对象，使用数据模板协议，用户的设备需按数据模板定义要求传输设备数据到云端，并可使用基于数据模板的诸多业务功能。

### 属性
属性是数据模板的一部分，属性是对物理设备运行状态的数据抽象，包括物理设备的开关、温度、工作模式、转速等表现设备当前运行状态的数据。这些数据具有只读或可操作的特点，对应应用系统发起的查看运行状态或设置设备运行状态的操作。

### 事件
事件是数据模板的一部分，事件是设备在某些场景下、某个时间设备运行状态的变化过程中需要主动将信息告知外部的抽象。包括设备发生的故障或者某个异常告警。

### 标准功能
标准功能是开发平台对接不同产品沉淀的默认功能集合，可为用户在定义产品数据模式时提供参考。

### 自定义功能
自定义功能是为用户提供个性化定义属性与事件的功能，用户可以参考标准功能，也可以根据设备的业务场景自由定义功能。

### MCU SDK
MCU SDK 是开发平台根据产品定义的数据模板与《腾讯云 IoT AT 指令协议》自动生成 MCU 程序框架，嵌入式开发工程师可基于 MCU SDK 快速通过腾讯云支持的通信模组对接到开发平台。

### App 开发
App 开发是开发平台提供的 App 应用开发能力，用户可通过此能力创建生成唯一的 AppKey 与 AppSecret，调用云端 API 完成 App 与设备的互动。

### 批量投产
用户的设备开发完成后，并在功能性与非功能性测试通过后，产品就会进入投产阶段，投产后产品定义的数据模板就不能修改，根据流程获取设备信息，并将数据烧录到投产的设备中。

### Topic
消息通信主题，Pub/Sub 模型中消息的通信媒介，发布订阅必须要有主题，通信时基于每个设备的具体 Topic。

### 发布
操作 Topic 的权限类型，对应的英文名称为 Pub，具有往 Topic 中发布消息的权限。

### 订阅
操作 Topic 的权限类型，对应的英文名称为 Sub，具有从 Topic 中订阅消息的权限。
