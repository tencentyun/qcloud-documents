## Overview 
The DNS hijacking detection feature can detect whether a domain name is hijacked and monitor the hijacking area distribution, the number of hijacked users and the hijacked IP address in real time.

## Configuration Example
### Preventing Home Page from Being Tempered
1. Log in to the [WAF Console](https://console.cloud.tencent.com/guanjia
), and click Website Security Detection -> **DNS Hijacking Detection** to enter the page of basic settings of DNS hijack detection.
![DNS](https://mc.qcloudimg.com/static/img/bfdea24d12433439b3de57cb88b44728/dns_01.png)
2. Click **Add Domain Name**, and the domain name information window pops up. 
Enter the domain name to be detected and the corresponding authorized IP address, and click **Save**.
If there are multiple authorized IP addresses, click **+ Add a Line** to add more IPs.
![DNS](https://mc.qcloudimg.com/static/img/be0f66895e0216ad7ea9b7fcd92f5659/dns_02.png)
3. After DNS hijacking detection is configured successfully, the relevant domain name records are shown on the basic configuration page.
![DNS](https://mc.qcloudimg.com/static/img/2f808769ab3909dc929eda47a1044115/dns_03.png)
