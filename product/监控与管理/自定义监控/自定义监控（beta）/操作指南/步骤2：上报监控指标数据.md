## CLI 方式上报数据

1. [安装 TCCLI 工具](https://cloud.tencent.com/document/product/440/34011)。
2. 配置 TCCLI：执行以下命令，分别配置 secretId、secretKey、region 和 output 格式。
```
tccli configure set secretId   您的 secretId
# tccli configure set secretId 0000000
tccli configure set secretKey  您的 secretKey
# tccli configure set secretKey 111111111
tccli configure set region $(curl http://metadata.tencentyun.com/latest/meta-data/placement/region 2>/dev/null) 
tccli configure set output json
```
 - secretId 和 secretKey：可参考 [访问密钥](https://cloud.tencent.com/document/product/598/37140) 获取，本文命令举例使用的 secretId 和 secretKey 为0000000和111111111。
 - region：可执行本文命令直接获取参数值，或在本文 [地域列表](#.E5.9C.B0.E5.9F.9F.E5.88.97.E8.A1.A8) 章节查看可选区域的对应取值。
 -  output：使用默认的 JSON 输出格式。
3. 上报监控数据：执行`tccli monitor PutMonitorData`命令进行数据上报。
以下为一个上报示例，您可以直接复制完成上报体验，也可按照您的需要参考 [输入参数说明](#.E8.BE.93.E5.85.A5.E5.8F.82.E6.95.B0) 对参数进行调整后进行自由上报。
```shell
tccli monitor PutMonitorData \
--AnnounceInstance $(curl http://metadata.tencentyun.com/latest/meta-data/local-ipv4 2>/dev/null) \
--AnnounceTimestamp $(date +%s) \
--Metrics '[{"MetricName": "err_count","Value": 10},{"MetricName": "post_count","Value": 20}, {"MetricName": "users","Value": 30}]'
```
4. 上报成功：如下返回 RequestId 则说明上报成功，上报成功后系统将自动注册指标，已注册指标不会重复创建。
```shell
{ "RequestId": "dc874995-e065-41a9-9aeb-bb02942d65e3"}
```
5. 您还可以编写 Shell 文件调用 TCCLI 工具完成数据上报，如下为一个 Shell 文件示例，您还可以参见最佳实践 [监控 Nginx 进程存活](https://cloud.tencent.com/document/product/397/40210) 文档查看如何通过更完整的 Shell+CLI 的方式完成数据上报。
```shell
#!/bin/bash
err_count=100
post_count=10000
users=99
metrics=$(cat <<EOF
[
  {
    "MetricName": "err_count", 
    "Value":$err_count 
  }, 
  {
    "MetricName": "post_count", 
    "Value": $post_count
  }, 
  {
    "MetricName": "users", 
    "Value": $users
  }
]
EOF
)
myip=$(curl http://metadata.tencentyun.com/latest/meta-data/local-ipv4 2>>/dev/null)
tccli monitor PutMonitorData --Metrics "$metrics" --AnnounceInstance "$myip"  --AnnounceTimestamp $(date +%s)
```

```go
package main

import (

				var resp *http.Response

        region, _ := ioutil.ReadAll(resp.Body)

                return;

				// 实例化一个认证对象，入参需要传入腾讯云账户 secretId，secretKey

				req.AnnounceTimestamp = proto.Uint64(uint64(time.Now().Unix()))

                {proto.String("err_count"), proto.Uint64(*err_count)},
                {proto.String("post_count"), proto.Uint64(*post_count)},
                {proto.String("users"), proto.Uint64(*users)},
        }
        
        // 发送 request 请求并响应
        rsp, err := client.PutMonitorData(req)
        if err != nil {
                log.Fatal(err)
        }
        log.Println(rsp.ToJsonString())

}
```

3. Python 语言 Demo

```python
# -*- coding: utf-8 -*-
import time
import urllib2
from tencentcloud.common import credential
from tencentcloud.common.exception.tencent_cloud_sdk_exception import TencentCloudSDKException
from tencentcloud.monitor.v20180724 import monitor_client, models

try:
   # 获取 region 地区
    url_region = "http://metadata.tencentyun.com/latest/meta-data/placement/region"
    region = urllib2.urlopen( urllib2.Request(url_region) ).read()
    #print(region)
    # 获取实例 IP
    url_ip = "http://metadata.tencentyun.com/latest/meta-data/local-ipv4"
    instance =  urllib2.urlopen( urllib2.Request(url_ip)).read()
    #print(instance)

    cred = credential.Credential("secretId", "secretKey")

    client = monitor_client.MonitorClient(cred, region)

    req = models.PutMonitorDataRequest()

    req.AnnounceInstance = instance
    req.AnnounceTimestamp = int(time.time())
    req.Metrics = [
      {"MetricName": "err_count","Value": 10},
      {"MetricName": "post_count","Value": 20},
      {"MetricName": "users","Value": 30}
    ]
    resp = client.PutMonitorData(req)
    print(resp.to_json_string())

except TencentCloudSDKException as err:
    print(err)
```

4. JAVA 语言 Demo

```java
package com.tencent.monitor;

import com.tencentcloudapi.common.Credential;
import com.tencentcloudapi.common.exception.TencentCloudSDKException;
import com.tencentcloudapi.common.profile.ClientProfile;
// 导入对应产品模块的 client
import com.tencentcloudapi.monitor.v20180724.MonitorClient;
// 导入要请求接口对应的 request response 类
import com.tencentcloudapi.monitor.v20180724.models.PutMonitorDataRequest;
import com.tencentcloudapi.monitor.v20180724.models.PutMonitorDataResponse;
import com.tencentcloudapi.monitor.v20180724.models.MetricDatum;

import java.util.Date;

public class PutMonitorData
{

	public static void main(String [] args) {
		try{
      
			// 实例化一个认证对象，入参需要传入腾讯云账户secretId，secretKey
			Credential cred = new Credential("secretId", "secretKey");

			// 实例化要请求产品（以 monitor 为例）的 client 对象
			ClientProfile clientProfile = new ClientProfile();
			clientProfile.setSignMethod(ClientProfile.SIGN_TC3_256);

			// （必选项）定义上报机器所在地域，可用地域参考https://cloud.tencent.com/document/api/213/15692文档
			MonitorClient client = new MonitorClient(cred, "ap-guangzhou", clientProfile);
			PutMonitorDataRequest req = new PutMonitorDataRequest();

			// [可选项] 定义上报的 IP (建议本机的内网 IP)，如果不填，将使用请求到云网关接口机最后一跳的外网 IP（可能是出口 IP 或者代理 IP)
			req.setAnnounceInstance( "127.0.0.1" );

			// [可选项] 定义上报的时间戳，如果不填，将使用服务器系统时间戳
			Long ts = new Date().getTime()/1000;
			req.setAnnounceTimestamp( ts.intValue() ); 
			MetricDatum[] metrics = new MetricDatum[3];
			String[] metricsName = new String[]{"err_count","post_count","users"};
			for(int i=0; i<3; i++) {
				metrics[i] = new MetricDatum();
				// 定义MetriName
				metrics[i].setMetricName(metricsName[i]);
				// 定义Value
				metrics[i].setValue(10);
			}

			req.setMetrics(metrics);
			// 发送 request 请求并响应
			PutMonitorDataResponse resp = client.PutMonitorData(req);
			System.out.println(PutMonitorDataResponse.toJsonString(resp));

		} catch (TencentCloudSDKException e) {
			System.out.println(e.toString());
		}
		
	}
}
```

## 上报接口说明

#### 接口描述

| 接口描述             | 接口说明                                                     |
| -------------------- | ------------------------------------------------------------ |
| 接口请求域名         | `monitor.tencentcloudapi.com`                                  |
| 默认接口请求频率限制 | 50次/秒                                                      |
| 上报指标限制         | 单次上报最多30个指标/值对，请求返回错误时，请求中所有的指标/值均不会被保存。 |
| 默认指标总数         | 100个/租户                                                   |
| 上报的时间戳         | 期望保存的时间戳，建议构造整数分钟时刻的时间戳。 时间戳时间范围必须为当前时间到300秒前之间。 同一 IP 指标对的数 据需按分钟先后顺序上报。 |

#### 输入参数

以下请求参数列表仅列出了接口请求参数和部分公共参数，完整公共参数列表见 [公共请求参数](https://cloud.tencent.com/document/product/248/30346)。

| 参数名称          | 必选 | 类型                 | 描述                                                         |
| ----------------- | ---- | -------------------- | :----------------------------------------------------------- |
| Action            | 是   | String               | 公共参数，本接口取值：PutMonitorData                         |
| Version           | 是   | String               | 公共参数，本接口取值：2018-07-24                             |
| Region            | 是   | String               | 公共参数，腾讯云设备报的地域信息（可以通过`http://metadata.tencentyun.com/latest/meta-data/placement/region` 获取），查询可用 [地域列表](#.E5.9C.B0.E5.9F.9F.E5.88.97.E8.A1.A8) |
| Metrics.N         | 是   | Array of MetricDatum | 一组指标和数据                                               |
| AnnounceInstance  | 否   | String               | 上报实例名或 IP（IP 可通过`http://metadata.tencentyun.com/latest/meta-data/local-ipv4`获取），最大长度128字节 |
| AnnounceTimestamp | 否   | Integer              | 上报时自行指定的时间戳，例如1572438660                      |

其中，MetricDatum 参数说明如下：

| 参数名称   | 必选 | 类型    | 描述                                                 |
| ---------- | ---- | ------- | ---------------------------------------------------- |
| MetricName | 是   | String  | 指标名称，最大长度255字节，只允许`A-Z`、`a-z`、`0-9`、`-_`字符匹配 |
| Value      | 是   | Integer | 指标值，整数，不得超过32字节，超过32字节会被截断     |

#### 输出参数

| 参数名称  | 类型   | 描述参数名称                                                 |
| --------- | ------ | ------------------------------------------------------------ |
| RequestId | String | 唯一请求 ID，每次请求都会返回。定位问题时需要提供该次请求的 RequestId |
