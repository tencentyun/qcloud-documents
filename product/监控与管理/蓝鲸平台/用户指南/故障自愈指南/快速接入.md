## 操作场景
下面将为您详细介绍如何快速使用故障治愈功能。

## 操作步骤
### 创建磁盘清理自愈套餐
1. 登录故障治愈平台。
2. 在左侧导航栏中，单击【接入自愈】>【套餐管理】，进入套餐管理页面。
3. 单击【创建自愈套餐】，配置以下选项。
 - 套餐类型：根据实际环境进行选择。
 - 套餐命名：输入套餐命名。
 - 磁盘清理的目录：输入需要清理的磁盘目录。
 - 删除多少天前的文件：选择天数。
 - 待删除文件名描述：匹配文件描述。
![](https://main.qcloudimg.com/raw/8f6c7f41b1c249cef98f7ade95367a73.png)
4. 配置选项之后，单击【保存自愈套餐】。



### 接入磁盘清理自愈方案

上述我们创建了磁盘清理自愈套餐，接下来我们让磁盘使用率告警接入这个套餐。
1. 在故障治愈平台左侧导航栏，单击【接入自愈】>【接入自愈】，进入接入自愈配置管理页面。
2. 在页面右上方，单击【接入自愈】，配置以下选项。
 **自愈场景**模块下配置：
  - 告警类型：将告警接入自愈。
  - 按内容筛选：可选， 一般在自定义告警下使用，匹配告警内容。
  
 **自愈处理**模块下配置： 
 - 是否启用：勾选是，标识启用该自愈套餐。
 - 自愈套餐：选择告警类型对应的自愈套餐。
 - 通知方式：选择自愈执行结果的通知方式。
 - 通知人员：选择自愈执行时的通知接收人。
![](https://main.qcloudimg.com/raw/8f12c53c3fed7ae939fd95c9a5f98cd8.jpg)
3. 配置上述选项之后，下拉页面，找到“额外信息”配置模块，配置以下选项。
 **超时**：如果自愈执行超过指定时间，则按失败处理，默认为40分钟。
![](https://main.qcloudimg.com/raw/a9810392e1d872d3f8d401238d975734.png)
4. 配置完成之后，单击【保存自愈策略】。



### 配置告警通知渠道
故障自愈是蓝鲸 PaaS 上一个 SaaS，通知渠道使用 PaaS 的通知 ESB 组件，在蓝鲸的独立部署版本（企业版、社区版）需要在开发者中心后台设置。
1. 打开“开发者中心”，单击【API 网关】，进入 API 网关管理界面，
2. 在页面上方菜单栏，单击【通道管理】，将“系统”选择为“[CSMI]蓝鲸消息管理”，找到消息通知相关，如下图所示。
![](https://main.qcloudimg.com/raw/f2e2958984fc88a9efe3beb21af1e016.jpg)
详细设置请访问蓝鲸 PaaS 提供的 [配置告警渠道](https://docs.bk.tencent.com/product_white_paper/paas/CaseScenario.html#noticeWay) 文档。
测试邮件服务是否正常，请参阅 [蓝鲸经验](https://bk.tencent.com/s-mart/community/question/95#/)分享文档 。
