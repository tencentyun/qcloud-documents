Redis 单机版是针对特定场景的版本，采用单个数据库节点部署架构，兼容 Redis 2.8 版本的协议和命令。与主从版本相比，只包含一个节点，数据只有一份，**不提供数据持久化和备份**，适用于数据可靠性要求不高的纯缓存业务场景。![](https://main.qcloudimg.com/raw/dba336b5ffa8b277bc2c36f3a8cd030d.svg)

## 功能特性
 - **低成本**
 Redis 单副本架构采用单节点部署。HA（High Availability）系统会定时探测节点的健康状况，如果发现业务不可用，HA 系统会在30秒内重新拉起一个 Redis 进程继续为用户提供 Redis 服务，服务可用性相比双副本版本并未下降。同时，通过减少一个数据库节点，可以大幅节省用户成本，售价约为双副本版本的一半。
 - **高性能**
由于双副本架构的备库仅用于故障转移恢复，并没有对外提供服务，并且数据库复制也给主库带来了额外的性能开销，因此单副本版本的性能不会低于双副本版本。

## 适用场景
 - **纯缓存场景** 
单副本版本只有一个数据库节点，节点出现故障时，系统会重新拉起一个 Redis 进程（**没有数据**），当节点故障业务自动切换完成后，应用程序需要将数据重新预热，以免对后端数据库产生访问压力冲击。
>!由于单副本版本不能提供数据可靠性，节点故障后需要业务进行预热，如果是对数据可靠性要求较高的敏感性业务，不建议使用单副本版本，可选用双副本版本。
 - **业务初期测试使用** 
业务开发、测试阶段，对数据可靠性要求不高，同时希望能降低成本时，可选择单机版，后续业务正式上线后换成主从版。

## 使用限制
-  不支持数据高可靠，当 Redis 节点故障后，系统会在一分钟内提供一个新的服务节点，原有数据将无法恢复，要业务重新预热数据。
-  单机版不提供数据备份，以及数据恢复功能。
