## 2.1	算术运算符 ##
您可以使用算数运算符对 TDSQL PostgreSQL 版（Oracle 兼容）所支持数据类型的值进行操作。本文介绍 TDSQL PostgreSQL 版（Oracle 兼容）支持的算数运算符。

下表展示了允许使用的算数运算符。

|运算符|	描述|示例|结果|
| ---- | ---- | ----- | ------- |
|+	|加法|	4 + 4|	8|
|-	|减法|	5 - 2|	3|
|*	|乘法|	4 * 4|	16|
|/	|除法|	16/4|	4.0000000000000000|
此处的除法与 Oracle 运算结果精度稍有差异，TDSQL PostgreSQL 版（Oracle 兼容）运算精度为小数点后 16~20 位，Oracle 的运算精度为小数点后最多 9 位，自动省略末尾 0，小于 1 时省略个位数。

#### 示例

 TDSQL PostgreSQL 版（Oracle 兼容）：

```
postgres=# select 16/4 from dual;
  ?column?  
--------------------
 4.0000000000000000
(1 row)
postgres=# select 2/3 from dual;
?column?
------------------------
 0.66666666666666666667
postgres=# select 4/4 from dual;
?column?
------------------------
 1.00000000000000000000
(1 row)
```

Oracle：

    SQL> select 16/4 from dual;
    
       16/4
    ----------
     4
    SQL> select 2/3 from dual;
    
       2/3
    ----------
    .666666667
    SQL> select 4/4 from dual;
    
       4/4
    ----------
     1
    SQL> select 20/8 from dual;
    
      20/8
    ----------
       2.5


## 2.2	连接运算符 ##
TDSQL PostgreSQL 版（Oracle 兼容）支持 Oracle 的 SQL 连接符 `||`，支持字符与字符、字符与数字、数字与数字类型的连接运算。

| 描述       | 示例       | 结果     |
| ---------- | ---------- | -------- |
| 字符与字符 | `'a'||'b'` | `ab`     |
| 字符与数字 | `'a'||1`   | `	a1` |
| 数字与数字 | ` 1||2`    | `12`     |
|            | `1||2.1`   | `12.1`   |
|            | `1.1||2.1` | `1.12.1` |

#### 示例

```
postgres=# select 'a'||'b' from dual;
 ?column? 
----------
 ab
(1 row)

postgres=# select 'a'||1 from dual;
 ?column? 
----------
 a1
(1 row)

postgres=# select 1||2 from dual;
 ?column? 
----------
 12
(1 row)

postgres=# select 1||2.1 from dual;
 ?column? 
----------
 12.1
(1 row)

postgres=# select 1.1||2.1 from dual;
 ?column? 
----------
 1.12.1
(1 row)
```


## 2.3	集合运算符 ##
TDSQL PostgreSQL 版（Oracle 兼容）兼容 Oracle 的集合运算符 UNION、UNION ALL、INTERSECT、MINUS。

|运算符	|描述|
|---|---|
|UNION| 多个 SQL 结果求并集，结果集去重 |
|UNION ALL	|多个 SQL 结果求并集，结果集不去重|
|INTERSECT	|多个 SQL 结果求交集，结果集去重|
|MINUS	|多个 SQL 结果求差集，结果集去重|

#### 语法

    SELECT STATEMENT
    UNION | UNION ALL | INTERSECT | MINUS
    SELECT STATEMENT
    [
    UNION | UNION ALL | INTERSECT | MINUS
    SELECT STATEMENT
    ... 
    ]
    [ ORDER BY [ number [ ASC | DESC ] , ... ]

#### 示例

    create table student
    (
    id int primary key,
    name nvarchar2(50) not null,
    score number not null
    );
    insert into student values(1,'Aaron',78);
    insert into student values(2,'Bill',76);
    insert into student values(3,'Cindy',89);
    insert into student values(4,'Damon',90);
    insert into student values(5,'Ella',73);
    insert into student values(6,'Frado',61);
    insert into student values(7,'Gill',99);
    insert into student values(8,'Hellen',56);
    insert into student values(9,'Ivan',93);
    insert into student values(10,'Jay',90);

UNION：

    postgres=# select * from student where id < 4 union select * from student where id > 2 and id < 6;
     id | name  | score 
    ----+-------+-------
      5 | Ella  |73
      3 | Cindy |89
      1 | Aaron |78
      2 | Bill  |76
      4 | Damon |90
    (5 rows)

UNION ALL：

    postgres=# select * from student where id < 4 union all select * from student where id > 2 and id < 6;
     id | name  | score 
    ----+-------+-------
      1 | Aaron |78
      2 | Bill  |76
      5 | Ella  |73
      3 | Cindy |89
      3 | Cindy |89
      4 | Damon |90
    (6 rows)

INTERSECT：

    postgres=# select * from student where id < 4 intersect select * from student where id > 2 and id < 6;
     id | name  | score 
    ----+-------+-------
      3 | Cindy |89
    (1 row)

MINUS：

    postgres=# select * from student where id < 4 minus select * from student where id > 2 and id < 6;
     id | name  | score 
    ----+-------+-------
      1 | Aaron |78
      2 | Bill  |76
    (2 rows)


## 2.4	其它运算符 ##
### 2.4.1	(+) ###
TDSQL PostgreSQL 版（Oracle 兼容）支持 Oracle 的 SQL（左/右）外连接 `(+)` 写法，等同于 `left | right join`。

#### 使用限制

关于使用 `(+)` 的注意事项：

1. `(+)` 操作符只能出现在 `where` 子句中，且不能与 `outer join` 语法同时使用。
1. 使用 `(+)` 操作符进行关联操作时，如果 `where` 子句包含多个条件，则关联的两张表的其它所有条件上关联表的字段上都必须包含 `(+)` 操作符。
1. `(+)` 操作符只能用于列，不能用于表达式。
1. `(+)` 操作符不能与 `or` 和 `in` 操作符一起使用。

#### 语法

左外连接（left join）：

```
... WHERE table1.column1 =table2.column1(+) [ AND table1.column2 =table2.column2(+) ...]
```

右外连接（right join）：

    ... WHERE table1.column1(+) =table2.column1[ AND table1.column2(+) =table2.column2 ...]

#### 示例

    create table a(a1 number(12),a2 varchar2(50));
    create table b(b1 number(12),b2 varchar2(50));
    insert into a values (1,'a'),(2,'ba'),(3,'ca'),(4,'da');
    insert into b values (1,'a'),(2,'bba'),(3,'ca'),(5,'dda'),(6,'eda');

左外连接（left join）：

    postgres=# select a.a1,b.b1 from a,b where a.a1=b.b1(+) ;
     a1 | b1 
    ----+----
      1 |  1
      2 |  2
      4 |   
      3 |  3
    (4 rows)

右外连接（right join）：

    postgres=#  select a.a1,b.b1 from a,b where a.a1(+)=b.b1;
     a1 | b1 
    ----+----
      1 |  1
      3 |  3
      2 |  2
    |  5
    |  6
    (5 rows)

### 2.4.2	空值与NULL ###

- 空值：即空字符串，两个单引号，中间无空格等任何内容。
- NULL：SQL关键字，代表没有。

Oracle 中的空值被当做 NULL 处理 ，而 PostgreSQL/TDSQL PostgreSQL 版中空值与NULL不同，TDSQL PostgreSQL 版（Oracle 兼容）则保持与 Oracle 一致。

下表是空值与 NULL 在不同数据库中的对比情况。

|描述|	Oracle	|PostgreSQL/TDSQL PostgreSQL 版| TDSQL PostgreSQL 版（Oracle 兼容） |
|--|--|--|--|
|空值与NULL是否一致|	一致|	不一致|	一致|
|NULL与数值运算	|NULL	|NULL	|NULL|
|空值与数值运算	|NULL|	报错	|NULL|
|空值排序	|空值最大|	空值最大|	空值最大|
|NULL排序	|NULL值最大|	NULL值最小|	NULL值最大|

#### 使用限制

1. 空值、NULL 与任何数值运算，结果为 NULL。
2. SQL中空值、NULL 不能用 `=`、`!=` 判断，应使用 `is null`、`is not null`。
3. 对于有空值、NULL 的情况，需要注意排序时，如有需要可以用 `nvl` 转换为数值。

#### 示例

TDSQL PostgreSQL 版（Oracle 兼容）

```
postgres=# select nvl2('','1','2') from dual ;
nvl2 
------
 2
(1 row)

postgres=# select 1 from dual where '' is null ;
 ?column? 
----------
1
(1 row)

postgres=# select null||'tbase' from dual;
 ?column? 
----------
 tbase
(1 row)

postgres=# select null+1 from dual;
 ?column? 
----------
 
(1 row)

postgres=# drop table t_null;
DROP TABLE
postgres=#  create table t_null(f1 int,f2 varchar2(10));
CREATE TABLE
postgres=#  insert into t_null values(1,null);
INSERT 0 1
postgres=#  insert into t_null values(2,'');
INSERT 0 1
postgres=# select * from t_null where f2 is null;
 f1 | f2 
----+----
  1 | 
  2 | 
(2 rows)

postgres=# select * from t_null where f2 ='' ;
 f1 | f2 
----+----
(0 rows)

postgres=# insert into t_null values(3,'1'); 
INSERT 0 1
postgres=# insert into t_null values(4,'2'); 
INSERT 0 1
postgres=# select * from t_null order by f2;
 f1 | f2 
----+----
  3 | 1
  4 | 2
  1 | 
  2 | 
(4 rows)
```
