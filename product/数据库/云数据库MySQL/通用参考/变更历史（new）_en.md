## Development History
### 2017
* Read-only instances supported elastic specifications without the need to be specified with the same specifications with that of master instances.
* Monitoring with the granularity of 60s was supporteed for CDB for MySQL
* Data subscription was supported for DTS
* Physical backup was supported for instances
* Manual backup initiation, custom backup time and storage period up to 732 days were supported.
* DTS data migration was compatible with various network environments
* CDB-CDB data migration was supported for DTS
* Data management console (DMC) was published
* Real-time monitoring and instance session management were supported for DMC
* Security group was supported for CDB


### 2016
* Read-only instance went online
* Postpaid instance was supported

### 2015
* High IO went online, with a performance 10 times higher than the previous Standard version
* Web rollback, and batch database management were supported
* Public network access and Web data import/export were supported
* CDB was renamed to CDB for MySQL

### 2014
* Multi-project for one user was supported for CDB; more billing methods were put into use
* Underlying storage was optimized with improved performance

### 2013
* 2nd generation high-performance version went online

### 2012
* High-performance version went online
* Standard version went online; MySQL kernel was optimized

### 2011
* 1st generation CDB went online; MySQL management feature was improved
