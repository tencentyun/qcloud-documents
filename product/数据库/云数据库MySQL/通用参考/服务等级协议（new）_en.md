## 1. Persistency of Data Storage

During the period of agreement, we guarantee a 99.9996% persistency for the data stored in the instances applied by users each month. That is, for each month, if a user stored 1,000 KB files in the instances, only 4 files are possibly lost.

## 2. Destroyable Data

When you request to delete any data or before you discard or resell any device, Tencent Cloud completely deletes all your data by performing a low-level formatting on the disk. The deleted data cannot be recovered and the disk is demagnetized when it is due for scrap.

## 3. Data Migration

Tencent Cloud provides data service in standard MySQL database file format, and allows users to store data as a standard SQL file using its import/export tool, thus helping users import data to cloud databases, or export data to their virtual hosts or local servers.

## 4. Data Privacy

Tencent Cloud implements network isolation by configuring firewall policy and using whitelist filtering mechanism, and ensures that the data of a user is not visible to any other users within the same resource pool by using the username and password for database instance as well as the permission control mechanism to disable database instances.

## 5. Right to Know

1. The location of data centers where the data is stored. You can make an inquiry about this by submitting a ticket.
2. The number of data backups and the location of data centers where the backup data is stored. You can make an inquiry about this by submitting a ticket.
3. Tencent Cloud helps you select the data center that is most suitable for your network environment to store data. Cold backup is dynamically allocated according to resource usage. By default, you are not required to choose data center and cold backup center location. If you need to do so, you can make an inquiry about this by submitting a ticket.
4. the PRC. You can make an inquiry about this by submitting a ticket.
5. Any of your data is not disclosed to any third party, unless such disclosure is required by regulatory authorities for supervision and auditing purposes. Your behavior logs are used for data analysis for keeping track of database status, but not used to disclose any of your personal information to any third party.

## 6. Data Auditing

In accordance with the applicable laws and regulations and on condition of compliance with relevant process and availability of all necessary documents, Tencent Cloud may provide information regarding cloud databases, including operation log of key components, operation records of OPS personnel and operation records of users, if required by regulatory authorities or if it is necessary to do so for other reasons such as collection of evidences during investigation into security incidents.

## 7. Business Feature

The service features supported for Tencent Cloud CDB instances include CDB instance application, login, addition/deletion/modification/query of database, cold backup data extraction, data rollback, data import, multi-thread data import/export tool, OPS data query API and password reset. All these features come with detailed descriptions and documentation. Users are notified of any change of each feature that can affect their data result via internal message.

## 8. Business Availability

1. We guarantee a 99.95% business availability for CDB, which means the service should be available for users for at least 99.95% of the time, or 30(day)*24(hr)*60(min)*99.95%=43,178.4 minutes, and be unavailable for 43,200-43,178.4=21.6 minutes at most each month. The statistics of service unavailability is calculated based on a single user instance.
2. If the business recovers from failure within 5 minutes, it is not counted into service downtime. Unavailability duration refers to the period from the moment the failure occurs to the recovery of service, including maintenance duration.

## 9. Resource Allocation Capability

Tencent Cloud guarantees that requests for scaling up computing resources by less than 50% from the current capacity for less than 10 instance resources is satisfied within 1 hour, for less than 30 instances, 24 hours, but for more than 30 instances, submit a ticket to inquiry the completion time. Each user can expand up to 100% capacity and a minimum capacity of 25 GB at a time.

## 10. Failure Recovery Capability

Tencent Cloud's professional team provides maintenance support on a 24/7 basis.
