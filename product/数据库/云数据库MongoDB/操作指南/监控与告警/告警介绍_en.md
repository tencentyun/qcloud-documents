Tencent Cloud MongoDB monitoring provides multi-dimensional custom alarm features which will send alarms via SMS when metric data exceeds certain thresholds

1. Go to **Cloud Monitoring** > **My Alarms** > **Alarm Policy"", and click **Add Alarm Policy**
![](https://mc.qcloudimg.com/static/img/33def73633ed5fe186d4f7c58a3477ae/g1.png)

2. Configure alarm rule and select MongoDB as the policy type
![](https://mc.qcloudimg.com/static/img/75c4f08032ec997362df3a1bae1b2dc0/g2.png)

3. Associate alarm object, that is, the instance object to be associated
![](https://mc.qcloudimg.com/static/img/0fc359a6696bd95f7292c23563573143/g3.png)


4. Set receiving group, that is, the target that will receive the SMS
![](https://mc.qcloudimg.com/static/img/5ecbcfe40b4a83c097bc22a8e4499389/g4.png)
