#一般型问题

## 什么是数据传输服务 DTS?
数据传输服务（Data Transmission Serivce，DTS）提供数据迁移、数据同步、数据订阅于一体的数据库数据传输服务，帮助您在业务不停服的前提下轻松完成数据库迁移，利用实时同步通道轻松构建异地容灾的高可用数据库架构。

## 如何开始使用腾讯云 DTS？
可以。您可以使用腾讯云 DTS 将数据一次性迁移到基于腾讯云数据库，也可以进行持续的数据复制。腾讯云 DTS 会捕获来源数据库的变更，并将它们以事务一致的方式应用到目标。如何进行持续的数据复制请参考 [数据订阅](https://cloud.tencent.com/document/product/571/8774)。

## DTS 是否能支持两个不同腾讯云账号下的 CDB 实例之间的数据迁移？
可以，进行跨腾讯云账号下 CDB 实例间的迁移，需要以目标 CDB 实例所属的腾讯云账号登录 DTS，同时源实例类型选择有公网 IP 的自建数据库。

## 腾讯云 DTS 支持哪些来源和目标？
腾讯云 DTS 目前支持来自有公网 IP 的 MySQL、云主机上自建的 MySQL、专线接入的 MySQL、VPN 接入的 MySQL、CDB for MySQL 的源数据库，以及 CDB for MySQL 的目标数据库。

## 可以监控数据库迁移任务的进度吗？
可以。您可以在腾讯云 DTS 控制台管理页面查看到迁移任务的进度。

## 使用腾讯云 DTS 进行数据迁移，源数据库的数据会不会在迁移后被删除掉？
不会，DTS 进行数据迁移时，其实是复制了源数据库的一份数据，并不会对源数据库的数据有任何影响。

## 腾讯云 DTS 是否支持定时自动迁移？
支持。您可以创建腾讯云 DTS 后，在修改配置时将定时执行的选项打钩，并在下方选择定时迁移时间。

# 操作流程问题

## redis 迁移适用于什么版本？
源实例版本大于 3.2 版本以及 3.2 版本，均无法迁移。

## mysql 迁移适用于什么版本？
可支持 MySQL 5.1/5.5/5.6 版本迁移上云；由于目前腾讯云 CDB 已不再支持 MySQL 5.1 版本，因此我们推荐您在迁移前完成 MySQL 5.1 升级到 MySQL 5.5，然后再迁移至 CDB for MySQL 5.5。当然您也可以选择使用 DTS 数据迁移工具直接从本地 MySQL 5.1 迁移至腾讯云 CDB for MySQL 5.5。

## 检测连通性失败是什么原因？
单击【详情点击】可查看解决方案。

## 目的实例为什么不可使用？
目的实例存在以下情况不可以使用：
1. 目的实例未初始化；
2. 目的实例被其他任务锁住 ；
3. 目的实例有数据；
4. 源实例使用数据大于目的实例目标实例容量。

## 为什么校验任务时发生警告？
可单击警告项目右侧 【查看详情】，可查看内容原因以及解决方案。
 
## 迁移过程中发生错误导致迁移任务失败是什么原因？
1. 迁移过程中源实例 bgsave 失败；
2. 迁移过程中源写量过大，超过设置的同步 BUFFER，导致同步连接重连，迁移任务会不断重试连接，生成 RDB。
