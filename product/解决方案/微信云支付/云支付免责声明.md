## 云支付收银终端
1. 使用云支付收银终端时，交易结果必须以云支付收银台为准，不能以其他第三方收银软件的任何行为及用户的支付渠道的凭证为准，例如收银软件出小票、收银软件提示支付成功等。
2. 云支付收银终端支付成功的标准如下：
 - 云支付收银终端的收银小窗口提示支付成功。
 - 若已配置云支付收银终端打印收银小票的功能，以成功打印收银小票为准。
 - 若已配置收款语音播报的功能，支付成功语音播报可能存在一定的延迟。务必要确认是本次支付的语音播报，否则不能作为支付成功的依据。
 - 收银小窗口未提示支付成功时，请在订单查询中确认订单的支付结果为支付成功。
3. 云支付收银终端退款受理成功的标准如下：
 - 云支付收银终端的退款小窗口提示了退款受理成功。
 - 若已配置云支付收银终端打印收银小票的功能，以成功打印退款小票为准。
 - 在退款小窗口未提示退款受理成功或退款小票打印失败的情况下，请在订单查询中确认订单的退款的结果为受理成功或退款成功。
>!未按照以上标准来判断交易结果时，云支付不承担任何责任。
4. 所有收银金额获取方式。在云支付收银终端发起支付时，收银员需要对系统识别出的收银金额进行确认，确认无误后，才能发起支付。
>!在收银设置中“扫码人工确认收银时间”和“快捷键确认收银时间”，建议设置为“扫码人工确认收银时间”。没有设置人工确认造成订单金额错误时，云支付不承担任何责任；已设置人工确认但收银员没有确认金额正确而造成订单金额错误的情况，云支付不承担任何责任。
5. 金额获取错误存在多种情况，包括但不仅限于以下情况：
 - 客显方式：收银软件修改了客显发送数据的方式，就可能造成金额获取错误。
 - 图像识别不能达到100%的识别正确率，服务商在选择图像识别时要多测试不同组合的金额是否能够正确识别，减少金额识别错误的风险，降低服务商自己的风险。
 - 收银员误操作导致金额识别错误。
 - 顾客提前扫付款码导致金额识别错误。
6. 若出现订单纠纷，需要提供云支付收银终端小票、云支付收银终端日志等证据，否则交易结果以云支付收银终端订单查询为准，云支付不予处理、不承担任何责任。  

## 配置支付宝授权
服务商在腾讯云控制台添加商户后，配置商户支付宝时，如果出现多次支付宝授权，且前后授权的支付宝商户实体不一致，造成商户资金损失，云支付不承担任何责任，由服务商承担责任。

## HTTPS API 接口
第三方对接云支付 HTTPS API 接口时，交易接口没有按照文档说明判断交易结果，造成商户资金损失的，云支付不承担任何责任，由服务商承担责任。

## 退款权限管理
服务商或商户对退款权限（包含支付退款、押金解冻等权限）管理不善，而造成商户资金损失的，云支付不承担任何责任。例如，服务商或商户离职人员随意操作退款等。

## 配置门店
服务商或商户在配置门店内的口碑门店、支付宝子商户信息、微信支付子商户号时，请仔细核对正确。因配置错误而造成商户资金损失时，云支付不承担任何责任。
